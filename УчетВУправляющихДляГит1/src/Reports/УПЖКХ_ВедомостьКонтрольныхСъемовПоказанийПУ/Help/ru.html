Отчет предназначен для формирования ведомости контрольных съемов показаний расчетных электрических счетчиков потребителей. Данная ведомость поддерживает как печать на бумажном носителе, так и выгрузку в *.xml файл.

Данная ведомость используется для обмена данными с системой «Контроль электрических приборов учета», разработанной компанией «Агент Плюс»: сформированный xml-файл передается и загружается в указанную систему.

Последующая загрузка показаний приборов учета, полученных от мобильной системы, осуществляется с помощью обработки «Загрузка реестра показаний счетчиков» 

Если счетчик не действует на начало предыдущего месяца указанной даты, на которую анализируются данные, то он не попадает в таблицу результатов.

 

Перед формированием отчета необходимо указать следующие параметры:

  • «Период» - дата формирования отчета, на которую анализируются данные;

 

Шапка отчета заполняется по данным справочника «Организации»:

  • В качестве имени организации указывается сокращенное наименование организации (значение поля «Сокращенное наименование (по учредительным документам)» на закладке «Основные»);
  • В качестве адреса указывается юридический адрес организации (значение поля «Юридический адрес» на закладке «Адреса и телефоны»).

 

Чтобы сформировать отчет необходимо нажать на кнопку «Сформировать».

 

Для того чтобы сохранить полученные данные в файл формата *.xml необходимо нажать на кнопку «Сохранить в файл». После нажатия на данную кнопку будет выведено диалоговое окно, где необходимо указать каталог расположения файла и его имя. Если сохранение файла прошло удачно, то в окно служебных сообщений будет выведено соответствующее сообщение.

Формат выгружаемого файла имеет следующую структуру:

<?xml version="1.0" encoding="utf-16">

<DATA DBVERSION="">

   <DOCUMENTS>

      <DOCUMENT GUID="">

         <ELEMENTS>

            <ITEM GUID="" DocNumber="" IsPost="" IsDeleted="" dt="" A01="" A02="" A03="" A04="" A05="" A06="" A07="" A08="" A09="" A010="" A011="" A012="" A013="" A014="" A015=""/>

         </ELEMENTS>

      </DOCUMENT>

   </DOCUMENTS>

</DATA>

где:

  • Элемент «DATA»:
    • Атрибут «DBVERSION» - Версия локальной базы данных мобильного устройства. Значение атрибута заполняется пустой строкой;
  • Элемент «DOCUMENT»:
    • Атрибут «GUID» - Идентификатор ведомости. Значение атрибута заполняется предопределенным значением «D18203A4-3458-47DB-8900-0B0E7233A45E»;
  • Элемент «ITEM»:
    • Атрибут «GUID» - Идентификатор счетчика;
    • Атрибут «DocNumber» - Номер документа. Значение атрибута заполняется пустой строкой;
    • Атрибут «IsPost» - Признак проведения документа. Значение атрибута заполняется «0»;
    • Атрибут «IsDeleted» - Признак удаления документа. Значение атрибута заполняется «0»;
    • Атрибут «dt» - Время создания элемента;
    • Атрибут «A01» - Номер лицевого счета;
    • Атрибут «A02» - ФИО жильца;
    • Атрибут «A03» - Состояние работы счетчика. Если счетчик включен, то значение атрибута заполняется «1», в противном случае «0»;
    • Атрибут «A04» - Улица;
    • Атрибут «A05» - Дом;
    • Атрибут «A06» - Квартира;
    • Атрибут «A07» - Корпус;
    • Атрибут «A08» - Тип счетчика. Значение атрибута определяется из заводского номера счетчика до первого символа «-»;
    • Атрибут «A09» - Номер счетчика. Значение атрибута определяется из заводского номера счетчика после первого символа «-»;
    • Атрибут «A010» - Значность. Значение атрибута принимает значение разрядности счетчика;
    • Атрибут «A011» - Номер пломбы счетчика. Значение атрибута заполняется пустой строкой;
    • Атрибут «A012» - Дата поверки счетчика;
    • Атрибут «A013» - Текущие показания счетчика (значение заполняется пустой строкой);
    • Атрибут «A014» - Количество проживающих (значение заполняется пустой строкой);
    • Атрибут «A015» - Примечание (значение заполняется пустой строкой).

 

Отчет открывается через меню «Учет в ЖКХ / Отчеты / Ведомость по ПУ».
