Документ «Регистрация оплаты»

Документ используется для регистрации оплаты за коммунальные услуги. Форма документа состоит из двух страниц: «Главное» и «Дополнительно».

В шапке документа располагаются следующие поля:

  • «Номер» – номер документа устанавливается автоматически;
  • «Дата» – дата документа устанавливается автоматически;
  • «Организация» – из справочника «Организации» выбирается организация;
  • «Вид операции» – выбирается вид операции документа; может принимать следующие значения:
    • «Поступление на банковский счет»;
    • «Поступление в кассу»;
    • «Поступление из автоматизированных систем приема оплаты»;
    • «Поступление с платежной карты»;  
    • «Возврат оплаты из кассы»;
    • «Возврат оплаты с расчетного счета».
  • «Источник оплаты» – из справочника «Источники оплат» выбирается источник оплаты;
  • «Здание» – из справочника «Здания, сооружения» выбирается здание;
  • «Не распределять оплату на добровольное страхование» – указывается вариант распределения оплат на добровольное страхование;

Примечание

Настройка доступна только в том случае, если ведется учет начислений по добровольному страхованию. Настройки ведения учета начислений по добровольному страхованию задаются в «Настройках параметров учета ЖКХ».

  • «Распределение на капремонт» – выбирается вариант распределения оплат на капитальный ремонт; может принимать следующие значения:
    • «Услуги, кроме кап. ремонта»;
    • «Услуги капитального ремонта».

    Примечание

    Поле доступно только в том случае, если для указанной в документе организации ведется раздельный учет по услугам капитального ремонта. Настройки ведения учета по услугам капитального ремонта задаются в соответствующем справочнике «Настройки учета по услугам капитального ремонта». При этом для вида операции «Поступление из автоматизированных систем приема оплаты» отражение типовых документов оплаты возможно только для варианта распределения «Услуги, кроме кап. ремонта».

В шапке документа также расположена гиперссылка «Настройки распределения оплат», при нажатии на которую открываются настройки распределения оплат «Учетной политики ЖКХ» по организации, указанной в шапке документа. Подробнее об общих настройках регистрации оплат смотрите в последних разделах настоящей справочной информации.

Страница «Главное»

На форме главной страницы присутствуют две табличные части:

  • Левая табличная часть состоит из колонок:
    • «N» – порядковый номер строки в табличной части;
    • «Лицевой счет» – из справочника «Лицевые счета» выбирается лицевой счет, по которому была произведена оплата услуг;
    • «Сумма» – сумма, на которую была произведена оплата услуг;
    • «Настройка распределения» – гиперссылка настройки распределения оплаты, при нажатии на которую открывается форма настройки текущей строки оплаты. Может принимать следующие значения:
      • «автоматически» - распределение будет выполнено автоматически в соответствии настройками «Учетной политики ЖКХ» организации;
      • «по услугам» - распределение будет выполнено по указанным услугам;
      • «по месяцам» - распределение будет выполнено по указанным месяцам;
    • «Документ оплаты» – выбирается документ, подтверждающий факт оплаты услуг;
    • «Помещение» – отображается помещение, соответствующее данному лицевому счету;
    • «Владелец» – отображается квартиросъемщик по данному лицевому счету;
    • «Сумма по графику» – сумма, которую необходимо оплатить в соответствии с графиком платежей.

    Над табличной частью документа, помимо стандартной панели работы с табличной частью, расположена кнопка «Заполнить», предназначенная для автоматического заполнения табличной части списком лицевых счетов указанного дома. Возможны следующие варианты автоматического заполнения: «Действующими лицевыми счетами» и «Действующими лицевыми счетами, имеющими задолженность».

    При помощи кнопки «Документы оплаты-Отразить оплаты» можно автоматически сформировать документы отражения оплат в строках табличной части, а при помощи кнопки «Документы оплаты-Очистить документы» можно автоматически очистить ссылки на документы отражения оплат во всех строках табличной части и пометить сами документы отражения на удаление. При этом для разных видов операции документа создаются различные типы документов оплаты. Для вида операции «Поступление на банковский счет» создаются документы «Поступление на расчетный счет»; для вида операции «Поступление в кассу» - «Поступление наличных»; для вида операции «Поступление из автоматизированных систем приема оплаты» -  «Корректировка долга»; для вида операции «Возврат оплаты из кассы» - «Выдача наличных»; для вида операции «Возврат оплаты с расчетного счета» - «Списание с расчетного счета»; для вида операции «Поступление с платежной карты» - «Операция по платежной карте». Вид операции формируемых документов отражения оплат зависит от вида договора лицевого счета (подробнее см. «Настройка параметров учета ЖКХ», закладка «Учет собственников»).

  • В правой табличной части выводится расшифровка оплаты по каждой строке левой табличной части. Табличная часть отображается в том случае, если на основной командной панели формы нажата кнопка «Расшифровка оплаты». Табличная часть расшифровки оплаты состоит из следующих колонок:
    • «Месяц/Услуга» – месяц и услуга, по которым была распределена оплата;
    • «Начисления-Сумма» – сумма, на которую была произведена оплата начислений в указанном месяце по указанной услуге;
    • «Начисления-Долг» – сумма задолженности по начислениям в указанном месяце по указанной услуге;
    • «Пени-Сумма» – сумма, на которую была произведена оплата пеней в указанном месяце по указанной услуге;
    • «Пени-Долг» – сумма задолженности по пеням в указанном месяце по указанной услуге;
    • «Рассрочка-Сумма» (отображается только в случае наличия задолженности по рассрочке) – сумма, на которую была произведена оплата рассрочки в указанном месяце по указанной услуге;
    • «Рассрочка-Долг» (отображается только в случае наличия задолженности по рассрочке) – сумма задолженности по рассрочке в указанном месяце по указанной услуге.

    На командной панели табличной части расположены кнопки автоматического распределения сумм:

    • «Распределить по текущей строке» – вызывает распределение суммы текущей строки таблицы оплат;
    • «Распределить по всем строкам» – вызывает распределение сумм всех строк таблицы оплат. Распределение представляет собой автоматическое заполнение таблицы расшифровки платежа по следующему правилу: формируется список начислений по текущему лицевому счету в разрезе услуг; сумма оплаты делится по полученным услугам пропорционально сумме начислений. Таким образом, расшифровка платежа заполняется разноской суммы оплаты по услугам, пропорционально начислениям по ним.

    При двойном нажатии на любую строку табличной части открывается форма редактирования расшифровки платежа, в которой можно вручную изменить распределение оплат.

Страница «Дополнительно»

На форме дополнительной страницы расположены следующие поля:

  • «Комментарий» – вводится комментарий к документу;
  • «Ответственный» – заполняется элементом справочника «Пользователи». Значение устанавливается автоматически текущим пользователем программы;
  • Табличная часть «Документы поступления денежных средств» - табличная часть доступна только для вида операции «Поступление на банковский счет». В строках таблицы можно указать документы «Поступления на расчетный счет», на основании которых вводятся оплаты;
  • Реквизиты оператора платежей (доступны только для вида операции «Поступление из автоматизированных систем приема оплаты»):
    • «Оператор» - из справочника «Контрагенты» выбирается оператор платежей;
    • «Договор оператора» - из справочника «Договоры» выбирается договор оператора платежей;
    • «Документ расчета с оператором» - из списка документов «Поступление на расчетный счет» выбирается документ расчета с оператором платежей, по которому оформляется оплата.

На основной командной панели формы в подменю «Печать» доступна кнопка «Печать расшифровки» для печати расшифровки оплаты (возврата).
На основной командной панели формы в подменю «Чеки» доступны кнопки для печати чеков в двух вариантах:

  • по всем строкам из левой табличной части документа. Для этого необходимо нажать кнопку «Напечатать по всем строкам».
  • по выделенной строке. Для этого необходимо нажать кнопку «Напечатать по текущей строке».


При проведении документ формирует движения по следующим регистрам:

  • «Взаиморасчеты по лицевым счетам»
  • «Начисления»
  • «Поступление денежных средств по видам платежей»
  • «Сведения о рассрочке»
  • «Начисления по добровольному страхованию»

 

Документ доступен в разделе интерфейса «Начисления услуг / Оплата / Регистрация оплаты».

Настройки учетной политики ЖКХ.

В «Учетной политике ЖКХ» организации имеются следующие настройки регистрации оплат:

  • на закладке «Регистрация оплаты» задаются настройки распределения оплаты, настройки распределения авансов, настройки считывания штрихкода, а также можно настроить порядок услуг, в соответствии с которым будет осуществляться распределение суммы оплаты.

  • в случае, если в организации применяется упрощенная система налогообложения, то доступна закладка «УСН», на которой задаются настройки отражения оплат в КУДиР.

Настройки параметров учета ЖКХ.

В «Настройках параметров учета ЖКХ» на закладке «Регистрация оплаты» задаются настройки печати кассовых чеков в платежных документах.
