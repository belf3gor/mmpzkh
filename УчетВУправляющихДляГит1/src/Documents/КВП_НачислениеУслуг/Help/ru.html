Документ «Начисление услуг»

Данный документ предназначен для начисления платежей за услуги ЖКХ по лицевым счетам. Выбор вида начисления осуществляется через меню «Операция». Возможные виды операций:

Начисление на лицевые счета
Начисления по показаниям приборов учета
Начисление рассрочки
Комиссия банка
Начисление добровольного страхования
Режим отладки расчетов

Вид операции «Начисление на лицевые счета»

Данный вид операции применяется при расчете услуг, которые не зависят от показаний приборов учета.

При вводе документа необходимо указать:

  • Номер – номер документа, устанавливается автоматически;
  • Дата – дата документа, устанавливается автоматически рабочей датой и может быть изменена. Дата документа определяет, за какой расчетный месяц будет производиться расчет платежей, и должна быть по времени позже даты назначения услуги на лицевые счета;
  • Организация – из справочника «Организации» выбирается организация, по которой необходимо начислить услуги;
  • Объект – указывается объект, по которому производится начисление платежей за услуги. В качестве объекта можно указать:
    • Здание или группу зданий;
    • Подъезд;
    • Помещение;
    • Лицевой счет или группу лицевых счетов.
  • Услуга – из справочника «Услуги» выбирается начисляемая услуга. Допускается не заполнять данное поле при данном виде операции. В этом случае в табличную часть попадут все назначенные услуги, за исключением услуг со способами расчета По показаниям счетчика и По показаниям счетчика и норме потребления;
  • Начислять за – количество дней, за которое будут начисляться услуги. Автоматически заполняется количеством дней в месяце расчета.

Табличная часть заполняется автоматически с помощью подменю Заполнить и одного из возможных вариантов заполнения:

  • Заполнить – при этом табличная часть заполняется всеми лицевыми счетами, по которым назначены услуги;
  • Заполнить с отбором – открывается обработка «Заполнение получателями услуг», с помощью которой можно заполнить документ только выбранными лицевыми счетами, удовлетворяющими установленному отбору.

Примечание
Если лицевой счет закрыт в месяце начисления, а основные начисления по нему уже были произведены, то при попытке повторного формирования документа "Начисление услуг" без отбора по услуге, расчет начислений по лицевому счету не производится.
В случае, если по закрытому лицевому счету необходимо провести доначисление услуги, можно воспользоваться одним из следующих способов:

  • перезаполнить и перепровести первоначальный документ начисления по данному лицевому счету;
  • оздать новый документ начисления услуг по данному лицевому счету с указанием услуге, начисление которой необходимо произвести.

    • Табличная часть состоит из колонок:

    • N – порядковый номер строки;
    • Лицевой счет – отображается лицевой счет, по которому произведены начисления. При автоматическом заполнении в табличную часть попадают только лицевые счета, по которым назначены услуги;
    • Услуга – отображается услуга, по которой произведены начисления;
    • Период – период, за который производится начисление. В качестве периода, как правило, берется весь месяц расчета. Но в некоторых случаях возможна разбивка на несколько подпериодов:
      • Если в течение периода начисления менялась площадь квартиры, то разбивка идет для услуг со способом расчета «По площади», «По нормам потребления» (если норматив зависит от площади), «С ограничением по нормам» (если составные услуги рассчитываются по площади).
      • Если в течение периода начисления менялось количество жильцов, то разбивка идет для услуг со способом расчета «По количеству жильцов», «По нормам потребления» (если норматив зависит от количества жильцов).
      • Если услуга назначена не с начала периода начисления, либо была приостановка действия услуги (разбивка будет производиться, если в настройках услуги не установлен флажок Начислять независимо от периода расчета).

      Пример: услуга «Антенна» со способом расчета «Фиксированное начисление» назначена на лицевой счет с 10.01.2014. Если флажок «Начислять независимо от периода расчета» не установлен, в документе за январь 2014 года период начисления будет 10.01.2014 – 31.01.2014. Если флажок установлен – 01.01.2014 – 31.01.2014.

      • Если в течение периода начисления менялся тариф по услуге.
      • Если в течение периода начисления менялся договор контрагента для взаиморасчетов.
    • Здание – отображается здание помещения, на которое открыт лицевой счет;
    • Квартира – отображается помещение, на которое открыт лицевой счет;
    • Владелец – отображается ответственный собственник, наниматель лицевого счета;
    • Количество – отображается количество потребления услуги. Количество зависит от способа расчета услуги.
      • Способ расчета Фиксированное начисление. Если на квартиру открыт только один лицевой счет, количество равняется 1. В случае если на квартиру открыто несколько действующих лицевых счетов (например, в случае долевой собственности), количество определяется исходя из настройки услуги Вариант распределения по л/с. Возможные варианты:
        • Полностью на каждый лицевой счет – для каждого лицевого счета количество будет равно 1;
        • Пропорционально долям лицевых счетов – количество будет распределяться пропорционально долям лицевых счетов;
        • Пропорционально количеству лицевых счетов в помещении – количество будет распределяться между лицевыми счетами поровну.
      • Способ расчета По количеству жильцов. Определяется количество жильцов (проживающих или зарегистрированных) по лицевому счету по состоянию на дату расчета. В случае если в течение периода начисления изменялось количество жильцов, то будет произведена разбивка по подпериодам и по каждому подпериоду будет указано реальное количество жильцов.
      • Способ расчета По площади. Определяется площадь помещения с учетом доли лицевого счета по состоянию на дату расчета. В случае если в течение периода начисления площадь помещения изменялась или изменялись доли лицевых счетов, будет произведена разбивка по подпериодам и по каждому подпериоду будет указана соответствующая площадь помещения.
      • Способ расчета По нормам потребления. Рассчитывается норматив потребления услуги, установленный для лицевого счета по состоянию на дату расчета. В случае если в течение периода начисления изменялись параметры, от которых зависит норматив, будет произведена разбивка по подпериодам и по каждому подпериоду будет указан соответствующий размер норматива.
      • Способ расчета С ограничением по нормам. В зависимости от способа расчета составных услуг выводится площадь помещения или количество жильцов (проживающих или зарегистрированных) по лицевому счету, при этом идет разбивка по услугам с учетом вида ограничения (например, по норме потребления и сверх нормы потребления).

        Пример: Лицевой счет открыт на квартиру с площадью 75 м2. На лицевой счет назначена услуга со способом расчета «с ограничением по нормам», при этом услуга в пределах норматива потребления 33 м2 рассчитывается по одному тарифу, свыше этого норматива – по другому тарифу. В документе начисления для данного лицевого счета произойдет разбивка по услугам: для услуги по норме количество будет равно 33, для услуги сверх нормы количество будет равно 42 (75-33), при этом каждая из услуг будет рассчитана по своему тарифу.

        Для лицевых счетов, площадь квартиры которых укладывается в норматив, начисление будет производиться только по услуге в пределах нормы.

    • Тариф – отображается тариф услуги, определенный на дату расчета. В случае разбивки начисления на подпериоды тариф определяется на конец каждого подпериода;
    • Начислено – отображается сумма, начисленная по данной услуге;
    • Дней начислено – отображается количество дней, за которое начислена услуга. В случае разбивки периода расчета на подпериоды количество дней будет считаться для каждого подпериода;
    • Договор контрагента – отображается договор с контрагентом. Определяется исходя из данных для взаиморасчетов, указанных при открытии или изменении лицевого счета. При этом если в течение периода начисления договор менялся, будут выведены несколько договоров.

    Под табличной частью выводится итоговая сумма начисления по документу.


      Начисление зависимых услуг


      Если выполнена настройка зависимых услуг, начисляемых в соответствии с объемами потребления услуг-оснований, то при автоматическом заполнении табличной части одновременно с основными услугами выполняется начисление по зависимым услугам (примером зависимых услуг является услуга водоотведение, начисляемая в соответствии с объемами потребления горячей и холодной воды).

      В этом случае в табличной части становится видимой колонка Услуга-основание, где отображается услуга, на основании которой произошло начисление зависимой услуги.

      Для удобства восприятия начисления основных и зависимых услуг в табличной части отображаются разными цветовыми схемами.

      ВАЖНО: данные начислений по зависимым услугам в строках по лицевому счету табличной части не суммируются с данными начислений услуг-оснований.

      Общую сумму начислений по зависимым услугам можно увидеть под табличной частью, которая выводится отдельно от суммы начислений по основным услугам.

      Примечание
      При ручном изменении объемов начисления услуг-оснований в табличной части автоматически изменяются объемы начислений и по зависимым от них услугам.


    Вид операции «Начисления по показаниям приборов учета»

    Данный вид операции применяется при расчете услуг, связанных с приборами учета (услуги должны иметь способы расчета По показаниям счетчика или По показаниям счетчика и норме потребления).

    При вводе документа необходимо указать:

    • Номер – номер документа, устанавливается автоматически;
    • Дата – дата документа, устанавливается автоматически рабочей датой и может быть изменена. Дата документа определяет, за какой расчетный месяц будет производиться расчет платежей, и должна быть позже даты назначения услуги и позже даты ввода показаний приборов учета (в случае их наличия);
    • Организация – из справочника «Организации» выбирается организация, по которой необходимо начислить услуги;
    • Объект – указывается объект, по которому производится начисление платежей за услугу. В качестве объекта можно указать:
      • Здание или группу зданий;
      • Подъезд;
      • Помещение;
      • Лицевой счет или группу лицевых счетов.
    • Услуга – из справочника «Услуги» выбирается начисляемая услуга. Заполнение поля обязательно для данного вида операции.

    Табличная часть заполняется автоматически с помощью подменю Заполнить и одного из возможных вариантов заполнения:

    • Заполнить – при этом табличная часть заполняется всеми лицевыми счетами, по которым назначены услуги;
    • Заполнить с отбором – открывается обработка «Заполнение получателями услуг», с помощью которой можно заполнить документ только выбранными лицевыми счетами, удовлетворяющими установленному отбору.

    После заполнения документа предлагается произвести расчет домовых начислений. Расчет можно выполнить, нажав кнопку «Рассчитать».

    Табличная часть состоит из колонок:

    • N – порядковый номер строки;
    • Лицевой счет – отображается лицевой счет, по которому произведены начисления. При автоматическом заполнении в табличную часть попадают только лицевые счета, по которым назначены услуги;
    • Услуга – отображается услуга, по которой произведены начисления;
    • Период – месяц расчета, за который производится начисление. При этом возможна разбивка на несколько подпериодов в следующих случаях:
      • Если в течение периода начисления установлены и/или отключены (по причине поломки или поверки) индивидуальные (квартирные) приборы учета.
      • Если услуга назначена не с начала месяца расчета, либо была приостановка действия услуги.
      • Если в течение периода начисления менялся тариф по услуге.
      • Если в течение периода начисления менялся договор контрагента для взаиморасчетов.
    • Вид тарифа – отображается вид тарифа, по которому произведены начисления. При этом в зависимости от тарифности приборов учета начисления по показаниям приборов учета производятся по дневному, ночному и пиковому видам тарифа, по нормативу потребления (при отсутствии показаний или действующих индивидуальных приборов учета) – по общему виду тарифа;
    • Здание – отображается здание помещения, на которое открыт лицевой счет;
    • Помещение – отображается помещение, соответствующее лицевому счету;
    • Ответственный собственник, наниматель – отображается ответственный собственник, наниматель лицевого счета;
    • Тариф – отображается размер тарифа соответствующего вида услуги, определенный по состоянию на дату расчета. В случае разбивки начисления на подпериоды тариф определяется на конец каждого подпериода;
    • Начисление при отсутствии ИПУ/показаний – в данной подгруппе колонок отображаются начисления при отсутствии индивидуальных приборов учета и/или показаний. Данная подгруппа колонок становится видимой в документе только в случае наличия подобных начислений хотя бы по одному из лицевых счетов.
      • Без ИПУ – отображается объем услуги по нормативу или по среднему значению при отсутствии индивидуальных (квартирных) приборов учета;
      • Без показаний – отображается объем услуги по нормативу или по среднему значению при отсутствии показаний индивидуальных (квартирных) приборов учета.

        В случае, если в предыдущем месяце расчета не были вовремя поданы показания и было произведено начисление при отсутствии показаний по нормативу или среднему расходу, а в текущем месяце расчета показания переданы, то осуществляется сторнирование предыдущих начислений при отсутствии показаний. При этом объем сторнировок отображается с минусом и выделяется красным;

      • Сумма – отображается сумма начислений без индивидуальных приборов учета и показаний;
    • Начисление по ИПУ – в данной подгруппе колонок отображаются начисления по показаниям индивидуальных приборов учета. Данная подгруппа колонок становится видимой в документе только в случае наличия начислений по показаниям индивидуальных (квартирных) приборов учета.
      • Объем – отображается объем услуги по показаниям индивидуальных (квартирных) приборов учета;
      • Сумма – отображается сумма начислений по показаниям индивидуальных (квартирных) приборов учета;
    • Учетный индивидуальный объем – информативное отображение индивидуального объема потребления услуги, который учитывается по лицевому счету при распределении ОДН. Данные, которые попадают в учетный индивидуальный объем, зависят от настроек услуги (поле В суммарный индивидуальный расход включать на закладке Настройки распределения по КПУ справочника Услуги).

      Пример: Суммарный расход по показаниям индивидуальных счетчиков составил 200 м3, суммарный расход по нормативу составил 100 м3, суммарный расход по среднему составил 150 м3. В зависимости от выбранной настройки услуги «В суммарный индивидуальный расход включать» в учетный индивидуальный объем попадет:

      • Если выбран вариант «Только расход по приборам учета», то в этом случае учетный индивидуальный объем составит 200 м3 (расход по нормативу и по среднему включены не будут);
      • Если выбран вариант «Расход по приборам учета и начисления по нормативу», то в этом случае учетный индивидуальный объем составит 200 м3 + 100 м3 = 300 м3 (расход по среднему включен не будет);
      • Если выбран вариант «Расход по приборам учета, начисления по нормативу и по среднему», то в этом случае учетный индивидуальный объем составит 200 м3 + 100 м3 + 150 м3 = 450 м3

      В объеме ОДН к распределению могут быть учтены объемы расхождений между начислениями при отсутствии показаний индивидуальных (квартирных) приборов учета и начислениями по фактическим показаниям индивидуальных (квартирных) приборов учета. Для этого в Настройках программы (ЖКХ) на закладке Приборы учета необходимо установить флаг Учитывать сторнирование индивидуальных начислений при расчете ОДН. При установленной настройке объемы расхождений учитываются в колонке Учетный индивидуальный объем.

      Пример: по лицевому счету "л/с №00001" начисление по услуге "Горячее водоснабжение" в силу отсутствия показаний прибора учета в январе было произведено по норме потребления в объеме 3 м3. При распределении ОДН был учтен данный объем. В феврале были поданы показания, объем потребления составил 5 м3. В этом случае в феврале производится сторнирование начисления по нормативу в размере 3 м3., а также начисление в размере 5 м3. В случае, если настройка «Учитывать сторнирование индивидуальных начислений при расчете ОДН» установлена, при распределении ОДН будет учитываться объем (учетный индивидуальный объем) в 2 м3. (5 – 3). Если настройка не установлена, при распределении ОДН будет учитываться объем в 5 м3.

      Сумма учетных индивидуальных объемов по лицевым счетам образует Суммарный индивидуальный объем, который отображается снизу на форме документа в поле Суммарный инд. объем.

      Из расхода по коллективному (общедомовому) прибору учета вычитается суммарный индивидуальный объем и определяется объем ОДН к распределению.

    • Распределение – в данной подгруппе колонок отображается распределение по лицевым счетам расходов на общедомовые нужды:

      • Кол-во по подъездам – отображается объем распределения ОДН по данным подъездного прибора учета (при отсутствии домового);
      • Кол-во по зданию – отображается объем распределения ОДН по данным общедомового прибора учета (при наличии и подъездных приборов учета, и домового, данные подъездных приборов учета не учитываются);
      • Сумма распределения – отображается сумма распределения ОДН.

      Подгруппа колонок Распределение отображается только при наличии распределения. Объем ОДН к распределению может быть как положительным, так и отрицательным. При этом распределяется всегда положительный объем ОДН. Отрицательный объем ОДН распределяется только в случае установленной настройки услуги Разрешить распределение отрицательного расхода ОДН.

      Объем распределения, который рассчитывается для лицевого счета, определяется исходя из настроек услуги (поле пропорционально).

      Пример: Имеется 3 лицевых счета: л/с №1 с площадью квартиры 75м2, л/с №2 с площадью квартиры 50 м2 и л/с №3 с площадью квартиры 90 м2. Объем ОДН к распределению составил 500м3. В настройках услуги определен способ распределения на лицевые счета пропорционально общей площади квартир. ОДН будет распределен следующим образом:
      л/с №1: 500 м3 × 75 м2 / 215 м2 = 171,419 м3;
      л/с №2: 500 м3 × 50 м2 / 215 м2 = 116,279 м3;
      л/с №3: 500 м3 × 90 м2 / 215 м2 = 209,302 м3.

      Примечание
      При распределение ОДН считается, что ПУ установлен, если он действовал хотя бы один день в периоде.

      Пример: По лицевому счету был установлен счетчик по услуге холодное водоснабжение, в середине периода он был отключен и новый установлен не был. В таком случае, считается, что по лицевому счету есть ПУ и ОДН распределяется(не распределяется в зависимостиот настройки услуги).

    • Начислено – отображается общая сумма, начисленная по услуге.
    • Договор контрагента – отображается договор с контрагентом. Определяется аналогично исходя из данных для взаиморасчетов, указанных при открытии или изменении лицевого счета. При этом если в течение периода начисления договор менялся, будут выведены несколько договоров.

    Снизу на форме документа отображаются итоговые показатели:

    • Расход по показаниям ИПУ – суммарный объем потребления по показаниям приборов учета по всем лицевым счетам (сумма объемов в колонке Начисление по ИПУ / Объем);
    • Расход при отсут. ИПУ / показаний – суммарный объем потребления при отсутствии счетчиков и при отсутствии показаний счетчиков по всем лицевым счетам (сумма объемов в колонках Начисление при отсутствии ИПУ/ показаний / Без ИПУ и Без показаний);
    • Суммарный объем сторно – суммарный объем сторнируемых начислений, рассчитанных ранее при отсутствии показаний;
    • Итоговый инд. объем (вкл. сторно) – общий суммарный объем индивидуальных начислений по всем лицевым счетам, включая объем сторнируемых начислений.
    • Расход по показаниям ОПУ – расход, определенный по показаниям коллективного (домового или подъездного) прибора учета. Определяется по данным, введенным в документе «Ввод показаний счетчика» с видом операции «Ввод показаний счетчика на здание» или «Ввод показаний счетчика на подъезд»;
    • Суммарный инд.объем – сумма учетных индивидуальных объемов лицевых счетов;
    • В т. ч. по общедомовым услугам – суммарный расход по общедомовым услугам, связанным через регистр сведений Услуги общедомового счетчика, индивидуальный расход по которым уменьшает коллективный расход по рассчитываемой услуге;
    • Объем ОДН к распределению – разница между расходом по общедомовому счетчику и суммарным индивидуальным объемом.

      Пример: Имеется услуга «Отопление», по которой установлен общедомовой прибор учета. Также имеется услуга «Подогрев», которая рассчитывается по показаниям приборов учета горячей воды. В регистре сведений «Услуги общедомового счетчика» определена зависимость между услугами: услуга общедомового счетчика – «Отопление», услуга-база расчета – «Подогрев».
      При начислении услуги «Подогрев» суммарный индивидуальный объем составил 300 м3.
      Показания общедомового счетчика по услуге «Отопление» составили 1000 м3. В документе «Начисление услуг» по услуге «Отопление» будет распределяться расход 1000 -300 = 700.
      Итоговые суммы будут следующие:
      Расход по показаниям ОПУ – 1000 м3;
      Суммарный инд. объем –300 м3;
         в т. ч. по общедомовым услугам – 300 м3;
      Объем ОДН к распределению – 700 м3 (1000 м3 – 300 м3).


      Начисление зависимых услуг


      Если выполнена настройка зависимых услуг, начисляемых в соответствии с объемами потребления услуг-оснований, то при автоматичеком заполнении табличной части одновременно с услугой-основанием выполняется начисление по зависимым от нее услугам (примером зависимых услуг является услуга водоотведение, начисляемая в соответствии с объемами потребления горячей и холодной воды).

      В этом случае в табличной части становится видимой колонка Услуга-основание, где отображается услуга, на основании которой произошло начисление зависимой услуги.

      Для удобства восприятия начисления основных и зависимых услуг в табличной части отображаются разными цветовыми схемами.

      ВАЖНО: данные начислений по зависимым услугам в строках по лицевому счету табличной части не суммируются с данными начислений основных услуг.

      Общую сумму начислений по зависимым услугам можно увидеть под табличной частью, которая выводится отдельно от суммы начислений по основным услугам.

      Примечание
      При ручном изменении объемов начисления услуг-оснований в табличной части автоматически изменяются объемы начислений и по зависимым от них услугам.

      Имеется возможность начисления зависимой услуги по собственному нормативу, в случаях, когда по всем услугам-основаниям отсутствуют действующие приборы учета. В соответствии с п.42 Постановления Правительства РФ № 354 «Размер платы за коммунальную услугу по водоотведению, предоставленную за расчетный период в жилом помещении, не оборудованном индивидуальным или общим (квартирным) прибором учета сточных бытовых вод, рассчитывается исходя из суммы объемов холодной и горячей воды, предоставленных в таком жилом помещении и определенных по показаниям индивидуальных или общих (квартирных) приборов учета холодной и (или) горячей воды за расчетный период, а при отсутствии приборов учета холодной и (или) горячей воды - исходя из норматива водоотведения».

      Для этого в настройки параметров учета ЖКХ добавлена возможность выбора способа начисления зависимых услуг:

      • В соответствии с объемами начисления услуг-оснований;
      • По собственному нормативу зависимой услуги.

      В случае выбора варианта «По собственному нормативу зависимой услуги» если хотя бы по одной из услуг-оснований зависимой услуги нет установленного индивидуального прибора учета, то начисление производится по нормативу потребления самой зависимой услуги. Для этого у лицевого счета по самой зависимой услуге должны быть установлены нормативы.

      ОСОБЕННОСТЬ: Начисление зависимых услуг по показаниям приборов учета / при отсутствии показаний приборов учета выполняется вместе с начислением услуг-оснований. Начисление же зависимых услуг по собственному нормативу, за те периоды, когда отсутствуют приборы учета, выполняется вместе с начислениями «обычных» услуг (с видом операции «Начисление на лицевые счета»). При этом при начислении услуг-оснований для зависимых услуг период, когда отсутствуют приборы учета, все равно выводится, но отмечается как начисление «по собственному нормативу» и начислений при этом не производится.

      Пример: Имеются услуги ХВС и ГВС, начисляемые по показаниям приборов учета, и услуга Канализация, зависимая от услуг ХВС и ГВС. В период с 01.09 по 05.09, например, производится замена ПУ (и по ХВС, и по ГВС) и приборы учета отключены. Для начисления услуг необходимо произвести:

      • 1) Начисление по показаниям приборов учета по услуге ГВС. При этом будет начислено:
        • ГВС с 01.09 по 05.09 - по нормативу потребления ГВС, с 06.09 по 30.09 - по показаниям ГВС.
        • Канализация с 01.09 по 05.09 - по собственному нормативу Канализации (расчет не выполняется, период выводится для информации), с 06.09 по 30.09 - по показаниям ГВС.
      • 2) Начисление по показаниям приборов учета по услуге ХВС. При этом будет начислено:
        • ХВС с 01.09 по 05.09 - по нормативу потребления ХВС, с 06.09 по 30.09 - по показаниям ХВС.
        • Канализация с 01.09 по 05.09 - по собственному нормативу Канализации (расчет не выполняется, период выводится для информации), с 06.09 по 30.09 - по показаниям ХВС.
      • 3) Начисление на лицевые счета по услуге Канализация.
        • Канализация с 01.09 по 05.09 - по собственному нормативу Канализации.


    Вид операции «Начисление рассрочки»

    Данный вид операции применяется для начисления услуг, на которые предоставлена рассрочка.

    При вводе документа необходимо указать:

    • Номер – номер документа, устанавливается автоматически;
    • Дата – дата документа, устанавливается автоматически рабочей датой и может быть изменена. Дата документа определяет, за какой расчетный месяц будет производиться начисление рассрочки;
    • Организация – из справочника «Организации» выбирается организация, по которой необходимо начислить услуги;
    • Объект – указывается объект, по которому производится начисление рассрочки. В качестве объекта можно указать:
      • Здание или группу зданий;
      • Подъезд;
      • Помещение;
      • Лицевой счет или группу лицевых счетов.

    Табличная часть заполняется автоматически с помощью команды Заполнить, расположенной в командной панели табличной части документа. При этом табличная часть автоматически заполняется по лицевым счетам, для которых предоставлена рассрочка.

    Табличная часть состоит из колонок:

    • N – порядковый номер строки;
    • Лицевой счет – отображается лицевой счет, по которому произведены начисления рассрочки;
    • Услуга – отображается услуга, по которой предоставлена рассрочка;
    • Заявление на рассрочку - ссылка на документ «Заявление на рассрочку», в рамках которого лицевому счету предоставлена рассрочка;
    • Договор - договор, используемый для взаиморасчетов с лицевым счетом;
    • Вид платежей - ссылка на перечисление "Виды платежей рассрочки" - вид платежей по рассрочке;
    • Месяц рассрочки - месяц начисления, за который производится начисление рассрочки;
    • Сумма остатка - сумма остатка долга по рассрочке на дату документа;
    • Сумма начисления - сумма начисления услуги по рассрочке;
    • Сумма комиссии - сумма комиссии за пользование рассрочкой.

    Вид операции «Комиссия банка»

    Данный вид операции предназначен для начисления комиссии банка. Используется для услуг, которые имеют способ расчета Комиссия банка.

    При вводе документа необходимо указать:

    • «Номер»» – номер документа, устанавливается автоматически;
    • «Дата»» – дата документа, устанавливается автоматически рабочей датой и может быть изменена.
    • «Организация»» – из справочника «Организации» выбирается организация, по которой необходимо начислить комиссию банка;
    • «Объект»» – указывается объект, по которому производится начисление оплаты за услуги. В качестве объекта можно указать:
      • Здание или группу зданий;
      • Подъезд;
      • Помещение;
      • Лицевой счет или группу лицевых счетов.
    • «Услуга» – из справочника «Услуги» выбирается начисляемая услуга со способом расчета Комиссия банка. Заполнение поля обязательно для данного вида операции.

    Табличная часть заполняется автоматически с помощью подменю Заполнить и одного из возможных вариантов заполнения:

    • Заполнить – при этом табличная часть заполняется всеми лицевыми счетами, по которым назначена услуга комиссии банка;
    • Заполнить с отбором – открывается обработка «Заполнение получателями услуг», с помощью которой можно заполнить документ только выбранными лицевыми счетами, удовлетворяющими установленному отбору.

    Табличная часть состоит из колонок:

    • N – порядковый номер строки;
    • Услуга – отображается услуга, по которой произведены начисления;
    • Лицевой счет – отображается лицевой счет, по которому произведены начисления. При автоматическом заполнении в табличную часть попадают только те лицевые счета, по которым была назначена услуга;
    • Здание – отображается здание помещения, на которое открыт лицевой счет;
    • Квартира – отображается помещение, на которое открыт лицевой счет;
    • Владелец – отображается ответственный собственник, наниматель лицевого счета;
    • Тариф – отображается тариф услуги (процент комиссии банка);
    • Сумма начислений – отображается сумма, начисленная в месяце расчета по услугам - базе для расчета начислений, определенных в услуге комиссии банка. От данной суммы производится расчет комиссии банка;
    • Начислена комиссия – отображается сумма рассчитанной комиссии банка. При этом комиссия может быть рассчитана двумя способами: В сумме от начислений за месяц и Сверху от суммы начисления за месяц. Выбор способа расчета комиссии осуществляется в настройках услуги в поле Расчет комиссии.

    Пример: Если процент комиссии банка установлен 3%, а услуги начислены на сумму 1603,87 р., то комиссия будет рассчитана следующим образом:

    • В сумме от начислений за месяц – начисление комиссии банка производится по формуле: S x St / (100 – St) , где S – сумма начислений за месяц, St – процент банка.
      Таким образом, комиссия будет равна 1603,87 р. × 3% / (100 – 3%) = 49,6 р.;
    • Сверху от суммы начисления за месяц – начисление комиссии банка производится по формуле: S x St / 100, где S – сумма начислений за месяц, St – процент банка.
      Таким образом, комиссия будет равна 1603,87 р. × 3% / 100 = 48,12 р.

    Вид операции «Начисление добровольного страхования»

    Данный вид операции предназначен для начисления услуги добровольного страхования имущества собственников помещений и доступен только при установленом флаге «Вести учет начислений по добровольному страхованию» в «Настройках параметров учета (ЖКХ)», на закладке «Учет начислений».

    При вводе документа необходимо указать:

    • «Номер»» – номер документа, устанавливается автоматически;
    • «Дата»» – дата документа, устанавливается автоматически рабочей датой и может быть изменена.
    • «Организация»» – из справочника «Организации» выбирается организация, по которой необходимо начислить добровольное страхование;
    • «Объект»» – указывается объект, по которому производится начисление оплаты за услуги. В качестве объекта можно указать:
      • Здание или группу зданий;
      • Подъезд;
      • Помещение;
      • Лицевой счет или группу лицевых счетов.
    • «Услуга» – услуга, которая является услугой добровольного страхования в соответствии с настройками, указанными в «Настройках параметров учета (ЖКХ)». Заполняется автоматически.

    Табличная часть заполняется автоматически с помощью подменю Заполнить и одного из возможных вариантов заполнения:

    • Заполнить – при этом табличная часть заполняется всеми лицевыми счетами, по которым назначена услуга добровольного страхования;
    • Заполнить с отбором – открывается обработка «Заполнение получателями услуг», с помощью которой можно заполнить документ только выбранными лицевыми счетами, удовлетворяющими установленному отбору.

     

    Табличная часть состоит из колонок:

    • N – порядковый номер строки;
    • Услуга – отображается услуга, по которой произведены начисления;
    • Лицевой счет – отображается лицевой счет, по которому произведены начисления. При автоматическом заполнении в табличную часть попадают только те лицевые счета, по которым была назначена услуга;
    • Квартира – отображается помещение, на которое открыт лицевой счет;
    • Владелец – отображается ответственный собственник, наниматель лицевого счета;
    • Количество – отображается количество потребления услуги;
    • Тариф – отображается тариф услуги;
    • Начислено – отображается сумма добровольного страхования, начисленная за месяц, указанный в шапке документа;
    • Дней начислено – отображается количество дней, за которое начислена услуга. Всегда равно полному количеству дней в месяце, указанном в шапке документа;
    • Договор - договор, используемый для взаиморасчетов с лицевым счетом.

    Режим отладки расчетов

    Данный режим позволяет быстро разобраться с выполненными начислениями услуг. При включенном режиме отладки становиться доступным и формируется во время начислении услуг протокол расчетов.еть

    Сейчас в протоколе расчетов фиксируется информация только при начислении услуг по показаниям ПУ, при этом в протокол расчетов выводятся сведения о том, как был произведен расчет учетного индивидуального объема и распределение СОИ.

    Режим отладки расчетов можно включить и отключить в документе «Начисления услуг» в панели расшифровки документа.

    Подробнее см. в инструкции.

    Внизу формы документа расположены кнопки:

    • ОК – провести документ и закрыть форму;
    • Записать – записать документ;
    • Закрыть – закрыть форму документа.

    Примечание
    При добавлении нового документа «Начисление услуг» копированием существующего табличные части, содержащие данные по начислениям, очищаются, так как для корректного формирования начислений требуется обязательное перезаполнение нового документа.

    Документ можно открыть через меню «Учет в ЖКХ - Начисление/Оплата - Начисление услуг».
