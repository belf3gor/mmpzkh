Исходящий электронный документ

Документ предназначен для хранения и отображения исходящих электронных документов.
Создать электронный документ можно двумя способами:

  • Из списков прикладных документов конфигурации, для которых предусмотрена возможность обмена электронными документами.
  • Из рабочего места Текущие дела по ЭДО.

В форме просмотра электронного документа отображаются:

  • Соответствующая печатная форма, заполненная на основании данных электронного документа. Такое представление позволяет визуально оценить содержимое электронного документа.
  • Установленные электронные подписи - подписи, которые уже установлены на документе.
  • Требуемые электронные подписи - требования к подписанию документа, которые нужно выполнить для завершения процесса подписания. Форма отображения аналогична отображению состава маршрута в справочнике Маршруты подписания.
  • Причины отклонения - содержит причины, по которым получатель электронного документа отказался от его подписания.
  • Причины аннулирования - содержит причины, по которым получатель электронного документа предложил его аннулировать.
  • Причины отказа от подписания - содержит причины, по которым один из подписантов вернул документ на доработку.
  • Сопроводительная записка - используется для передачи вместе с документом произвольной текстовой информации.

Основные доступные действия
  • В зависимости от настроек системы, текущего статуса документа и прав текущего пользователя в форме можно выполнить команды по переходу на следующий этап (Подписать, Подписать и отправить).
  • Пока документ находится в процессе подписания дополнительно для любого из текущих подписантов доступна команда Другие действия - Отклонить, с помощью которой можно отказаться от подписания документа и вернуть его на доработку.
  • Если помимо документа нужно передать какую-то дополнительную информацию, можно создать на его основании произвольный электронный документ с помощью команды Другие действия - Создать произвольный документ и приложить к нему необходимые файлы.

Работа с аннулированием

В процессе электронного документооборота может возникнуть потребность аннулирования конкретного документа (например, если допущена ошибка).
Для этого на стороне инициатора аннулирования используется команда Другие действия - Аннулировать.
На противоположной стороне в это случае используются команды:

  • Согласиться с аннулированием - документ аннулируется. Документ инициатора аннулирования при этом автоматически аннулируется после отправки соответствующей информации.
  • Не согласиться - документ возвращается в исходное состояние (до поступления предложения об аннулировании). На стороне инициатора аннулирования происходит то же самое после автоматической отправки отказа от аннулирования.

Печать электронного документа для предоставления во внешние инстанции на бумажном носителе

Распечатать электронный документ можно через Еще — Печать электронного документа.
Электронный документ будет выведен на печать в том виде, который виден на экране с дополнительным листом, где будут перечислены установленные электронные подписи, если такие есть.
Если электронный документ необходимо предоставить во внешние инстанции в электронном виде, то необходимо воспользоваться командой Еще — Сохранить электронный документ в файл.

Дополнительные возможности
  • Чтобы показать или скрыть информацию о связанных электронных документах нужно активировать опцию Скрыть, показать панель регламента на главной панели действий.
  • Чтобы изменить ответственного по документу, можно воспользоваться кнопкой Перенаправить.
  • В печатной форме документа можно отобразить область для заверения копии. Для этого можно воспользоваться командой Еще - Отображать область "Копия верна".
  • Для просмотра истории изменения документа можно открыть Журнал событий ЭДО из меню Еще.
  • Проверить статусы электронных подписей можно с помощью команды Проверить электронные подписи в меню Еще.
  • Если возникли проблемы с электронным документом, то можно предоставить необходимые данные для технической поддержки, выгрузив их в файл с помощью команды Сохранить электронный документ в файл в меню Еще.
  • Если подписанный электронный документ, оформленный ранее, требуется перенаправить получателю повторно, например, когда произошла потеря данных, то в форме документа можно воспользоваться командой Еще - Отправить повторно. В этом случае система отправит ранее созданный электронный документ со всеми установленными ЭП к моменту повторной отправки.
  • Для просмотра всех реквитов электронного документа можно воспользоваться командой Еще - Карточка документа.
  • Электронные документ может содержать в себе некоторую дополнительную информацию, обычно предназначенную для упрощения распознавания данных на принимающей стороне. При желании ее можно отобразить при помощи команды Еще - Отображать дополнительную информацию.

Особенности работы с произвольными электронными документами

Программа позволяет обмениваться Произвольными электронными документами — аналогами электронных писем. Произвольный электронный документ — это любой файл, хранящийся на компьютере. Он может создаваться на основании прикладных документов информационной базы, других электронных документов и непосредственно из формы Архив ЭДО.
Такой документ в отличие от других видов электронных документов не имеет однозначной структуры, поэтому работа с ним несколько отличается от работы с другими видами документов:

  • В произвольном документе до момента отправки на подпись можно вручную указывать организацию, договор, контрагента.
  • Если в Профилях настроек ЭДО или Настройках ЭДО для произвольного документа указан маршрут Указывать при создании документа, вручную указывается также маршрут подписания. При этом можно выбрать маршрут из справочника или непосредственно составить список подписантов, каждый из которых должен будет установить на документе электронную подпись.
  • Пока вложение не подписано, его можно заменить с помощью команды Заменить файл. Вложение может быть как добавлено с диска, так и выбрано из вложений основания произвольного документа.
