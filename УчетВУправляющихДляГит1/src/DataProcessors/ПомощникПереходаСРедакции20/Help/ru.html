Помощник перехода с редакции 2.0

После перехода с редакции 2.0 на редакцию 3.0 требуется восстановить последовательную нумерацию документов. Если этого не сделать, то для вновь создаваемых документов нумерация начнется с единицы. Помощник позволяет сделать это в автоматическом режиме.

Помощник содержит несколько шагов.

Префикс информационной базы

Если в редакции 2.0 был установлен префикс информационной базы длиной 3 символа, то при обновлении на новую редакцию он изменится. Он будет образован из первых двух символов старого префикса.

Новый префикс информационной базы не должен совпадать с префиксами во всех остальных базах, участвующих в синхронизации данных. Если это не так, то его нужно изменить.

Если префикс информационной базы после обновления на редакцию 3.0 не изменился, то этот шаг помощника будет пропущен.

Префиксы организаций

Длина префиксов организаций в редакции 3.0 была уменьшена до двух символов. Новый префикс организации образован из первых двух символов старого. Префикс организации не должен совпадать с префиксами остальных организаций.

В списке отображены все организации, у которых в редакции 2.0 был указан префикс.

Если префикс после обновления изменился, то он будет помечен зеленым цветом. Если префиксы нескольких организаций совпадают, то они будут отмечены красным цветом.

Если в списке есть несколько строк, которые отмечены красным, то переход на следующий шаг помощника будет заблокирован. Для продолжения работы помощника нужно скорректировать префиксы организаций так, что бы они были уникальны. После этого можно будет приступить к восстановлению нумерации.

Если при обновлении префиксы организаций не будут изменены, то этот шаг помощника будет пропущен.

Восстановление последовательной нумерации

При восстановлении последовательной нумерации будут созданы служебные документы с номером, который равен номеру последнего введенного в редакции 2.0 документа, но соответствует структуре номера редакции 3.0.

Служебные документы будут созданы для каждого вида документа по каждой организации. Дата документа будет установлена на 1 января текущего года. Документ не содержит данных и не должен проводится. Они предназначены исключительно для восстановления нумерации.

Пример:
В редакции 2.0 последнему введенному документу "Реализация товаров, услуг" был присвоен номер "00000000131". При восстановлении нумерации будет создан документ с номером "0000-000131" и датой 01.01.2014. После этого при вводе нового документа "Реализация товаров, услуг" автоматически установится правильный номер "0000-000132".
Последовательная нумерация документов будет сохранена.
