Проведение документов

Способ выполнения расчетов

В программе расчет стоимости списания МПЗ, зачет авансов и погашение задолженности при расчетах с контрагентами может выполняться двумя способами:

1) При закрытии месяца.

Этот способ позволяет значительно сократить время, требующееся для проведения документов. При выборе этого варианта в момент проведения первичных учетных документов (актов, накладных, банковских, кассовых и др.):

 - Не будут вычисляться суммы проводок по списанию себестоимости (проводки вида Дт ХХ - Кт 10, 21, 41, 43, 45). Проводки формируются по количеству и плановой себестоимости (если не задана, то по нулевой стоимости). Окончательные суммы списания рассчитываются регламентной операцией закрытия месяца "Корректировка стоимости номенклатуры".

 - Не формируются проводки по зачету авансов и погашению задолженности при расчетах с контрагентами по договорам в рублях (проводки вида Дт 62.02 - Кт 62.01 и др. аналогичные проводки при расчетах с поставщиками, прочими дебиторами и кредиторам). Эти проводки будут сформированы в ходе актуализации расчетов с контрагентами. Запустить актуализацию расчетов с контрагентами можно:

  • из окна просмотра движений документа;
  • из стандартных бухгалтерских отчетов (ОСВ по счету, Анализ счета, Карточка счета и др.), сформированных по счетам расчетов с контрагентами;
  • из рабочего места закрытия месяца, регистрации счетов-фактур на аванс и др., для работы с которыми необходимы данные о взаиморасчетах с контрагентами.

При расчетах с контрагентами по договорам в иностранной валюте либо в условных единицах, а также при расчетах по договорам с комитентами (принципалами) и комиссионерами (агентами) зачет авансов и погашение задолженности выполняется всегда сразу при проведении документов, независимо от выбранного способа расчетов.

Такой способ расчетов поддерживается:

 - для организаций, применяющих общую систему налогообложения, кроме ведущих раздельный учет входящего НДС;

 - для организаций и индивидуальных предпринимателей, применяющих УСН с объектом налогообложения "доходы" либо только патентную систему налогообложения и не использующих функциональность оплаты платежными картами.

Для организаций и индивидуальных предпринимателей, применяющих иные системы налогообложения либо в учетной политике которых определен способ оценки МПЗ "ФИФО" либо способ оценки товаров в рознице "По продажной стоимости", всегда используется способ "При проведении документов".

2) При проведении документов.

При выборе этого способа суммы проводок по списанию себестоимости, а также зачету авансов и погашения задолженности при расчетах с контрагентами рассчитываются сразу в момент проведения документов. Такой способ расчетов требует существенных затрат времени для вычисления остатков по счетам в процессе проведения документов.

Разрешается списание запасов при отсутствии остатков по данным учета

Если при проведении документов списания МПЗ на складе товары (материалы, готовая продукция) по данным учета отсутствуют, то будет выдано сообщение о нехватке и документы не будут проведены.

Для отключения этого контроля необходимо установить флажок "Разрешается списание запасов при отсутствии остатков по данным учета".

Внимание! Настройка действует только при выбранном способе расчетов "При проведении документов". Для способа "При закрытии месяца" контроль остатков не выполняется, проверить отсутствие отрицательных остатков на складе можно с помощью отчета "Контроль отрицательных остатков".

Проверять документы при формировании отчетов

Для организаций и ИП, применяющих УСН, ЕНВД или патентную систему, при формировании отчетов программа может автоматически проверять и актуализировать данные учета: восстанавливать последовательность проведения документов и закрывать все нужные месяцы в фоновом режиме.

Данные считаются актуальными при соблюдении двух условий:

  1. Должна быть соблюдена последовательность проведения документов, которая влияет на корректность расчета стоимости списанных запасов, правильность отражения взаиморасчетов с контрагентами и т.д.
  2. В конце каждого месяца должны быть выполнены регламентные операции (выполнено закрытие месяца): начислена амортизация, рассчитана себестоимость продукции, определен финансовый результат и т.п.

При формировании отчетов программа проверяет выполнение этих условий.

По результатам проверки программа предлагает актуализировать данные: последовательно выполнить перечисленные операции в фоновом режиме (кнопка Актуализировать).

Организациям, применяющим ОСНО, следует использовать Помощник закрытия месяца (Операции - Закрытие периода).

Время документов устанавливать автоматически

При установленном флаге документы в течение дня записываются со строго определенным временем. Конкретное время документа зависит от его вида. Это позволит избежать лишних оборотов по зачету авансов и отрицательных остатков в течение дня.
