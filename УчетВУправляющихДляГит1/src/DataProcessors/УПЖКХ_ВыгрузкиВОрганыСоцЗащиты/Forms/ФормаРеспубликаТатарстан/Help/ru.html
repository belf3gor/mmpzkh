Республика Татарстан

 

Обработка предназначена для выгрузки данных в отдел социальной защиты населения Республики Татарстан (далее - ОСЗН) за указанный период и загрузки начисленных в ОСЗН сумм льгот.

 

Перед началом загрузки или выгрузки данных в обработке необходимо заполнить следующие параметры:

  • «Период» – отчетный период.

  • «Организация» – ссылка на элемент справочника «Организации», по которой будут браться необходимые данные за указанный период.

  • «Префикс лицевых счетов» – префикс, стоящий в наименовании лицевых счетов перед номером (например, если наименование лицевого счета "л/с №7000201101", то здесь префикс "л/с №").

Используется для корректного срабатывания механизма идентификации лицевых счетов в базе при загрузке файлов "Ответ поставщику" (справочник ПСС) и "Начислено сумм по жильцам" (далее – входящие файлы).

  • «Группа лицевых счетов» – группа элементов справочника «Лицевые счета», по которой будет производиться выгрузка.

Если значение не заполнено, то выгрузка будет производиться по всем имеющимся лицевым счетам. Данный параметр используется только при выгрузке.

 

Настройка

Также перед началом выгрузки или загрузки данных необходимо произвести настройку обработки. Для вызова диалогового окна настройки необходимо нажать на кнопку «Настройка» на форме обработки.

 

Выгрузка данных в ОСЗН

При выгрузке данных в ОСЗН формируется файл установленного формата версии 2.6.

Формирование файла выгрузки данных в ОСЗН происходит на вкладке «Выгрузка данных в ОСЗН» основной формы. На этой вкладке необходимо указать файл "Ответ поставщику" (далее  – справочник ПСС). Справочник ПСС ежемесячно поставляется ОСЗН в формате «.crp» и перед выбором его предварительно нужно расшифровать.

При нажатии кнопки «Загрузить» произойдёт загрузка данных из справочника ПСС в нижележащую таблицу. При загрузке одновременно срабатывает механизм идентификации лицевых счетов в базе по соответствующему номеру домохозяйства в справочнике ПСС.

Если по каким–либо причинам идентификация лицевого счета не прошла, то будет выведено соответствующее сообщение с указанием номера строки, а сама строка с данным номером домохозяйства будет подсвечена серым цветом. Для данной строки можно указать лицевой счет в базе вручную, либо оставить эту строку не идентифицированной. В таком случае указанный ПСС не будет привязан ни к какому лицевому счету при выгрузке.

Примечание: Выгрузка может происходить вообще без загрузки справочника ПСС, однако, в виду отсутствия связей между ПСС и лицевыми счетами в базе, в выгружаемом файле значения полей ПСС не будут заполнены!

Перед непосредственным формированием файла выгрузки данных в ОСЗН необходимо указать каталог выгрузки в поле «Каталог выгрузки» и имя выгружаемого файла с расширением в поле «Имя файла».

Примечание: если имя выгружаемого файла не будет указано, то оно сформируется автоматически в процессе выгрузки в формате [Период.txt].

Для формирования файла выгрузки данных в ОСЗН необходимо нажать кнопку «Выгрузить».

Выгружаемый файл формируется в текстовом формате DOS (модифицированная альтернативная кодировка ГОСТ). Каждый файл состоит из строк. Признак окончания строки – пара символов с кодами 13 и 10 (перевод строки). Каждая строка состоит из полей. Признак окончания поля – символ "|". В значении полей символы "|" и символы с кодами 13 и 10 недопустимы. Каждое поле имеет тип – Текст(длина), Дата(ДД.ММ.ГГГГ), Целое, Число(<18>.<2>). Текущий формат выгружаемого файла соответствует версии 2.5.

Загрузка начисленных сумм льгот в ОСЗН

При загрузке начисленных сумм льгот в ОСЗН в базе формируется документ «Начисление услуг в сторонней программе» (далее – документ начисления).

Для проведения документа начисления необходимо заполнение следующих полей таблицы начисления документ начисления:

  • Услуга – услуга для загрузки льгот из ОСЗН (см. настройки загрузки начислений ОСЗН);

  • Лицевой счет – текущий лицевой счет;

  • Раздел учета – программно устанавливается значение «Льготы» (в таком случае при проведении такого документа формируется одна проводка по регистру накопления «Начисленные льготы»);

  • Льготник – жилец, имеющий право на получение субсидий и льгот, и относящийся к текущему лицевому счету;

  • Льготная категория – льготная категория в соответствии с кодом статьи начисляемой льготы ОСЗН (см. Настройки загрузки начислений ОСЗН);

  • Сумма – сумма начисленной в ОСЗН льготы.

Эти данные формируются автоматически при записи в документ начисления идентифицированных данных из файла "Начислено сумм по жильцам" (далее – файл начислений).

 

Загрузка начисленных сумм льгот в ОСЗН происходит на вкладке «Загрузка начисленных сумм льгот в ОСЗН» основной формы. На этой вкладке необходимо указать файл "Начислено сумм по жильцам". Файл начислений ежемесячно поставляется из ОСЗН в формате «sz_nach.j» и перед выбором его предварительно нужно распаковать (этот файл представляет собой архив в формате JAR 1.02 Copyright (c) 1996 –97 ARJ Software, Inc.). При нажатии кнопки «Загрузить» произойдёт загрузка данных из файла начислений в нижележащую таблицу. При загрузке одновременно срабатывают механизмы идентификации:

  • лицевых счетов в базе по соответствующему номеру домохозяйства;

  • льготников (жильцов) в базе по фамилии, имени и отчеству;

  • льготной категории в базе по коду статьи начисленных льгот (см. настройки загрузки начислений ОСЗН).

Если по каким–либо причинам идентификация по какому –либо из этих параметров не прошла, то будет выведено соответствующее сообщение с указанием номера строки, а сама строка будет подсвечена серым цветом. Для данной строки можно указать лицевой счет, льготника (жильца) и льготную категорию вручную. Крайне нежелательно оставлять строки не идентифицированными, так как в таком случае создаваемый документ начисления невозможно будет провести! Если какая–либо из строк не должна загружаться, то её необходимо удалить из текущей таблицы.

Для формирования документа начисления необходимо нажать кнопку «Записать». Запись может производиться как с созданием нового документа, так и в существующий не проведенный документ «Начисление услуг в сторонней программе». В последнем случае его необходимо выбрать в поле «Начисление услуг».
