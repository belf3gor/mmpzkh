Томская область

Обработка предназначена для выгрузки данных в отдел социальной защиты населения Томской области за указанный период.
 
Перед началом выгрузки данных в обработке необходимо заполнить следующие параметры:
  • «Организация» - ссылка на элемент справочника «Организации», по которой будут браться необходимые данные за указанный период;
  • «Период» - месяц, за который необходимо предоставить данные;
  • «Код организации» - указывается код Организации, согласно файлу organiza.dbf, либо вводится вручную;
  • «Каталог выгрузки» - указывается каталог для выгружаемого файла;
 
Перед началом выгрузки для корректного заполнения табличных частей обработки необходимо произвести предварительную настройку. Для вызова диалогового окна настройки необходимо нажать на кнопку «Настройка».

Перед началом установки настроек на самой форме настроек необходимо заполнить поле «Каталог загрузки» (указывается каталог загружаемых словарей area.dbf, place.dbf, streets.dbf, usluga.dbf).

Словари должны иметь следующую структуру:

  • area.dbf

    Наименование поля

    Значение поля

    code_area Код района (подрайона)
    area_name Наименование района (подрайона)
    owner Код района для подрайона
  • place.dbf

    Наименование поля

    Значение поля

    code_area Код района
    code_place Код населенного пункта
    place_name Наименование населенного пункта
  • streets.dbf

    Наименование поля

    Значение поля

    code_area Код района
    code_place Код населенного пункта
    code_stree Код улицы
    street_nam Наименование улицы
  • usluga.dbf

    Наименование поля

    Значение поля

    code Код жилищно-коммунальной услуги
    name Наименование жилищно-коммунальной услуги

Если структура файлов не будет соответствовать структуре описанной в таблицах, то выгрузка данных будет невозможна. Примечание: Словари organiza.dbf, urblag.dbf и dom.dbf при работе выгрузки не задействуются; коды элементов из этих словарей должны соответствовать кодам элементов базы данных.

На каждой из вкладок панели необходимо нажать кнопку «Загрузить из файла», в результате чего заполняться соответствующие таблицы данными из БД, либо нужно ввести данные вручную. Внимание! Вводимые в ручную данные имеют не ссылочный, а строковый (числовой для кодов) тип, поэтому следует иметь в виду что, например, записи "г.Томск", "Томск г." и "Томск г" будут считаться разными объектами.
На панели "Благоустройство объектов" имеется несколько групп: "Виды благоустройств" и "Свойство в базе". В группе "Виды благоустройств" определяется соответствие видов благоустройств "Наличие электрического нагревателя для отопления" и "Наличие газового котла" видам благоустройств из справочника "Виды благоустройства объектов". В группе "Свойство в базе" определяется дополнительный реквизит здания "Категория благоустройства" (этот реквизит должен быть добавлен на вкладке "Дополнительная информация" элемента справочника "Здания"), согласно которому будет определяться категория благоустройства здания.
На панели "Настройка соответствия услуг" необходимо установить соответствие ЖКУ, загруженных из файла, услугам из справочника "Услуги" в таблице "Услуги в базе". Примечание. Услуга из справочника "Услуги" не может соответствовать более чем одной услуге из файла .dbf. На панели "Услуги "Отопление"" необходимо указать услуги из справочника "Услуги", которые соответствуют услуге "Отопление" в ОСЗН.

Для осуществления выгрузки данных из табличной части необходимо нажать на кнопку «Выгрузить». Выгрузка данных в обработке происходит в файл gcusl.dbf со следующей структурой:
