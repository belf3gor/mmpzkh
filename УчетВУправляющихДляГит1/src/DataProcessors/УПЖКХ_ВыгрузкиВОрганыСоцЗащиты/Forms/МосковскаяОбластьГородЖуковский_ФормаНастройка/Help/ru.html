Московская область (г. Жуковский): Настройка

Вкладка «Настройка имен полей выгружаемых файлов»:

В таблице «Настройка имен полей выгружаемых файлов» предоставляется возможность изменять имена и регулировать состав полей выгружаемых файлов.

В колонке «Структура таблицы выгрузки» отображаются заданные по умолчанию в программе структуры таблиц выгружаемых файлов. Данные структуры представлены в виде двухуровнего иерархического списка: на верхнем уровне указано имя выгружаемой таблицы, на нижнем — имена полей выгружаемых файлов.

Для редактирования имени поля выгружаемого файла необходимо:

  • Перейти на нижний уровень соответствующей структуры таблицы выгрузки

  • Для редактируемого поля в колонке «Имя поля выгружаемого файла» установить необходимое имя. При этом, строка с измененным именем поля будет подсвечиваться бирюзовым цветом.

Для исключения поля из выгружаемого файла необходимо:

  • Перейти на нижний уровень соответствующей структуры таблицы выгрузки

  • Для исключаемого поля в колонке «Имя поля выгружаемого файла» очистить его имя. При этом, строка с исключенным из выгружаемого файла полем будет подсвечиваться серым цветом.

Нажатие на кнопку «По умолчанию» производит сброс всех установленных соответствий.

Вкладка «Соответствие видов услуг»:

На данной вкладке расположены две таблицы – «Виды услуг в ОСЗН» и «Услуги в базе». Выделяя какую-либо услугу в таблице «Виды услуг в ОСЗН» необходимо установить ей в соответствие одну и более услуг в базе в таблице «Услуги в базе».

При добавлении соответствий в таблице «Услуги в базе» необходимо придерживаться следующих правил:

·         Одному виду услуги в ОСЗН может соответствовать несколько услуг в базе.

·         Номер, указанный в поле «Приоритет», позволяет разделять услуги в базе, установленные в соответствие текущему виду услуг в ОСЗН, на группы с приоритетом учета тарифа по данным услугам в базе для расчета итогового тарифа по текущему виду услуг в ОСЗН. При этом, для расчета итогового тарифа берутся тарифы услуг в базе группы с наивысшим приоритетом, а если итоговый тариф, рассчитанный для этой группы услуг не удалось рассчитать (по этим услугам в базе нет начислений), то производится попытка расчета итогового тарифа по группе услуг в базе с более низким приоритетом.
По умолчанию у всех добавляемых услуг в базе приоритет «1».

·         «Тип услуги» отражает факт учета индивидуальной или общедомовой составляющей начисленной услуги. Если выбрать "Индивидуальная", то по данной услуге будут браться только индивидульаные начисления. Если выбрать "Общедомовая", то будут браться начисления ОДН. Если выбрать "Индивидуальная и общедомовая", то будут браться начисления в зависиомости от способа расчета услуги.

·         Флаг «Учитывать при расчете тарифа» отражает факт учета тарифа по текущей услуге в базе при расчете итогового тарифа по текущему виду услуг в ОСЗН.
По умолчанию этот флаг установлен всегда.

·         Флаг «Использовать нормировочный коэффициент» отражает факт расчета по текущей услуге в базе приведенного тарифа при наличии зависимости данной услуги от какой-либо услуги-основания. Приведенный тариф рассчитывается как произведение нормировочного коэффициента, равного отношению значения услуги–основания к значению зависимой услуги, согласно соответствующей записи в регистре сведений «Зависимости услуг», и текущего тарифа услуги в базе. Необходимость расчета приведенного тарифа через нормировочный коэффициент связана с тем, что при расчете итогового тарифа необходимо, чтобы единицы измерения тарифов подчиненных услуг были приведены к единицам измерения тарифов услуг-оснований.

Пример расчета приведенного тарифа. Пусть в базе существует две услуги:

o    «ХВС для ГВС» (единица измерения м3, тариф 20,18 руб./м3),

o    «Подогрев ХВС для ГВС» (единица измерения Гкал, тариф 1838,68 руб./Гкал),

И между этими услугами установлена следующая зависимость в регистре сведений «Зависимости услуг»:

Услуга-основание:

ХВС для ГВС

Значение основание:

0,088

Услуга:

Подогрев ХВС для ГВС

Значение:

1,000

Таким образом, услуга «Подогрев ХВС для ГВС» является зависимой от услуги «ХВС для ГВС» и на каждый кубометр затраченной воды приходится 0,088 Гкал энергии на подогрев.
В данном случае нормировочный коэффициент получается равным 0,088 Гкал / 1 м3 = 0,088 Гкал/м3. Умножая этот нормировочный коэффициент на тариф по услуге «Подогрев ХВС для ГВС» получаем приведенный тариф 0,088 Гкал/м3 × 1838,68 руб./Гкал = 161,80 руб./м3. То есть, полученный приведенный тариф сколько стоит подогрев каждого кубометра воды.

По умолчанию этот флаг всегда снят.

Нажатие на кнопку «По умолчанию» производит сброс всех установленных соответствий.

Вкладка «Соответствие видов благоустройств»:

В таблице «Соответствие видов благоустройств» необходимо произвести настройку соответствия видов благоустройств в ОСЗН и элементов справочника «Виды благоустройства объектов», либо установить в соответствие видам благоустройства в ОСЗН фиксированное число (в таком случае данная настройка будет действовать на все объекты). Примечание: одному виду благоустройства в ОСЗН может соответствовать только одна настройка соответствия.

Нажатие на кнопку «По умолчанию» производит сброс всех установленных соответствий.

Вкладка «Настройки выгрузки»:

На данной вкладке расположены следующие настройки:

·         «В колонку «Номер лицевого счета» выводить» - настройка вывода данных по лицевым счетам, а именно: какой реквизит справочника «Лицевые счета» будет выводиться в колонке «Номер лицевого счета» таблицы начислений;

·         «Выводить данные только по льготникам» - признак вывода данных только по льготникам (т.е. по тем открытым лицевым счетам, в которых зарегистрированы льготники) или по всем открытым в базе лицевым счетам.

·         «Способ свертки тарифа». Если одному виду услуг в ОСЗН соответствует несколько услуг в базе, отмеченных флажком «Учитывать при расчете тарифа», то данная настройка задает способ расчета итогового тарифа по этим услугам для данного вида услуг в ОСЗН.
