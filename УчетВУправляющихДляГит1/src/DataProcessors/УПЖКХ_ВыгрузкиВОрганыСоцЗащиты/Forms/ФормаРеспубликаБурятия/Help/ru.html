Республика Бурятия

Обработка предназначена для выгрузки данных в отдел социальной защиты населения Республики Бурятия.

 

Перед началом загрузки данных в обработке необходимо заполнить следующие параметры:

  • «Период» - отчетный период;
  • «Организация» - ссылка на элемент справочника «Организации», по которой будут браться необходимые данные за указанный период;
  • «Код организации» - числовой код, выводимый в поле «NJ» файла выгрузки.
  • Установить настройку «В поле «LS» выводить:». Для этого необходимо выбрать один из следующих пунктов списка
    • «Идентификатор» – в поле «LS» выводить значение реквизита «Идентификатор» элемента справочника «Лицевые счета»;
    • «Код» – в поле «LS» выводить код элемента справочника «Лицевые счета»;
    • «Наименование» – в поле «LS» выводить наименование элемента справочника «Лицевые счета»;
    • «Наименование (только цифры)» – в поле «LS» выводить цифровую часть наименования элемента справочника «Лицевые счета».

Настройка

Также перед началом выгрузки данных необходимо произвести настройку обработки. Для вызова диалогового окна настройки необходимо нажать на кнопку «Настройка» на форме обработки.

 

Выгрузка данных в ОСЗН

Перед выгрузкой необходимо заполнить таблицу выгрузки данными о взаиморасчетов нажатием кнопки "Заполнить". Заполненную таблицу можно выгружать в файл выгрузки для предоставления его в ОСЗН нажатием кнопки «Выгрузить». Файл выгрузки при этом будет сохранен в каталоге, указанном в поле «Каталог выгрузки», формат наименования файла «NJ_[Код организации].DBF».

Формат файла выгрузки:

Наименование поля

Наименование колонки

Тип

Значение заполнения

NJ

Код организации

Строка(10)

Код организации

LS

Лицевой счет

Строка(10)

Значение соответствующего реквизита выгружаемого лицевого счета

FAM

Фамилия

Строка(15)

Если ответственный собственник/наниматель выгружаемого  лицевого счета –  жилец, то выводить фамилию жильца.
Иначе выводить наименование контрагента выгружаемого лицевого счета

IMIA

Имя

Строка(15)

Если ответственный собственник/наниматель текущего лицевого счета –  жилец, то выводить имя жильца.

В противном случае не заполнять.

OTCH

Отчество

Строка(15)

Если ответственный собственник/наниматель текущего лицевого счета –  жилец, то выводить отчество жильца.

В противном случае не заполнять.

DATNACH

Отчетный период

Строка(6)

Отчетный период в формате «ГГГГММ».

SOSTSEM

Количество проживающих

Число(10,0)

Количество жильцов выгружаемого лицевому счету, проживающих по состоянию на конец отчетного периода.

ZHILPLOS

Площадь

Число(10,2)

Общая площадь помещения, приходящаяся на выгружаемый лицевой счет по состоянию на конец отчетного периода.

Для каждой услуги в ОСЗН выводить следующую группу колонок (здесь N – код услуги в ОСЗН):

TAR[N]

Тариф по услуге [Услуга в ОСЗН]

Число(10,2)

Размер общего (дневного) вида тарифа по услуге в базе, соответствующей услуге в ОСЗН с кодом [N] (далее – начисленная услуга), по которой на отчетный лицевой счета в отчетном периоде произведены начисления.
Если услуге в ОСЗН соответствует несколько начисленных услуг, то выводить  максимальное значение среди них.

POTR[N]

Потребление по услуге [Услуга в ОСЗН]

Число(10,5)

Значение, рассчитываемое как отношение значения, указанного в поле SUM[N] к значению, указанному в поле TAR[N].

Если значение в поле TAR[N] равно 0, то выводить значение, указанное в поле SUM[N].

SUM[N]

Начислено по услуге [Услуга в ОСЗН]

Число(10,2)

Фактическая сумма начислений, произведенных на выгружаемый лицевой счет в отчетном периоде (с учетом перерасчетов, без учета льгот, оплат и пеней) по начисленной услуге, соответствующей услуге в ОСЗН с кодом [N].
Если услуге в ОСЗН соответствует несколько начисленных услуг, то выводить сумму значений по ним.

OPL[N]

Оплачено по услуге [Услуга в ОСЗН]

Число(10,2)

Сумма оплат поступивших от выгружаемого лицевого счета за отчетный период по начисленной услуге, соответствующей услуге в ОСЗН с кодом [N].

Если услуге в ОСЗН соответствует несколько начисленных услуг, то выводить  сумму значений по ним.
