Республика Хакасия

Обработка предназначена для выгрузки данных в отдел социальной защиты населения республики Хакасия и загрузки данных о начисленных льготах за указанный период.

Перед началом загрузки или выгрузки данных в обработке необходимо заполнить следующие параметры:

  • «Период» - отчетный период;
  • «Организация» - ссылка на элемент справочника «Организации», по которой будут браться необходимые данные за указанный период;
  • «Префикс лицевых счетов» - префикс, стоящий в наименовании лицевых счетов перед номером (например, если наименование лицевого счета "л/с №7000201101", то здесь префикс "л/с №"). Используется для корректного срабатывания механизма идентификации лицевых счетов в базе при загрузке файлов "Ответ поставщику" (справочник ПСС) и "Начислено сумм по жильцам" (далее - входящие файлы).
  • «Льготная категория для загрузки из ОСЗН» - льготная категория в базе, по которой будет производиться загрузка сумм начисленных в ОСЗН льгот. Данный параметр используется только при загрузке. Рекомендуется для загрузки льгот в ОСЗН создать в базе новую отдельную льготную категорию, предназначенную только для этого механизма. При этом устанавливать эту льготную категорию жильцам в базе и устанавливать размер скидки по ней не нужно.
    Пример отдельной льготной категории:
    • Наименование - "Льгота для загрузки (ОСЗН)";
    • Полное наименование - "Льгота для загрузки (ОСЗН)".

Настройка

На форме настроек выгрузки имеются две вкладки "Таблица выгрузки" и "Таблица загрузки". Вкладка "Таблица выгрузки" предназначена для выгрузки данных о начислениях в файл выгрузки для предоставления его в ОСЗН. Вкладка "Таблица загрузки" предназначена для загрузки данных о льготах, начисленных в ОСЗН, в базу и создания на основе этих данных документа "Расчёт льгот" (по старому алгоритму расчета льгот).

Перед началом работы с выгрузкой необходимо на вкладках "Таблица выгрузки" и "Таблица загрузки" выбрать файл выгрузки и файл загрузки соответственно.

Примечание: Только при наличии выбранных файлов выгрузки и загрузки на форме настроек будет корректно сформирован список услуг в ОСЗН, т.к. этот список считывается непосредственно из файлов при их открытии!

Перед заполнением табличных частей обработки необходимо установить предварительные настройки. Для открытия формы настройки необходимо нажать кнопку «Настройка», расположенную в верхней части обработки.
Откроется форма настроек, которую необходимо заполнить следующим образом:

  • На вкладке «Настройки соответствия услуг в ОСЗН и услуг в базе» необходимо установить соответствия между элементами справочника «Услуги» с видом услуг в ОСЗН. При нажатии кнопки «Загрузить» левая колонка заполнится всеми видами услуг в ОСЗН, имеющимися в файлах загрузки. 
    Примечание: Для одной и той же услуги в ОСЗН (нижнего уровня) нельзя установить соответствие с одной и той же услугой в базе дважды (и более раз).
  • На вкладке «Настройки печати реестров стоимости ЖКУ» установить соответствия между полями реестров стоимости ЖКУ и видами услуг в ОСЗН.

Общие реквизиты данной обработки заполняются по умолчанию и при желании могут быть изменены:

  • «Организация» - ссылка на элемент справочника «Организации», по которой будут браться необходимые данные за указанный период;
  • «Период» - месяц, за который необходимо предоставить данные; 
    Примечание: период определяется также и при загрузке какого-либо файла. Поэтому рекомендуется перезагружать таблицы выгрузки/загрузки при соответствующей смене режима работы выгрузки, либо отслеживать корректность введённого периода вручную.
  • «Префикс лицевых счетов» - префикс, стоящий в наименовании лицевых счетов перед номером (например, если наименование лицевого счета "л/с №7000201101", то здесь префикс "л/с №"). Используется для корректного срабатывания механизма идентификации лицевых счетов в базе при загрузке DBF-файла.

Выгрузка данных в ОСЗН

В режиме выгрузки на вкладке "Таблица выгрузки" необходимо загрузить данные из выбранного файла выгрузки в обработку нажатием кнопки "Загрузить". При загрузке автоматически устанавливается соответствие данных файла данным в базе. Соответствие происходит путём определения лицевого счета в текущих данных. Если в строке лицевой счет в автоматическом режиме определить не удалось (о чём будет выдано сообщение), то такая строка будет подсвечена серым цветом. Пользователю предоставляется возможность установить соответствие вручную, выбрав лицевой счет в колонке "Лицевой счет в базе" "серой" строки.
Примечание: Строки с неопределенным лицевым счётом не участвуют в механизме выгрузки - данные о начислениях по ним заполнены не будут, однако эти строки останутся в файле выгрузки после его заполнения.

После загрузки необходимо заполнить таблицу выгрузки данными из файла нажатием кнопки "Заполнить". Заполненную таблицу можно выгружать в файл выгрузки для предоставления его в ОСЗН нажатием кнопки «Выгрузить», а также основываясь на ней напечатать реестры стоимости ЖКУ нажатием кнопки «Печать реестров».

Загрузка начисленных сумм льгот в ОСЗН

Предварительно на вкладке "Таблица загрузки" необходимо загрузить данные из выбранного файла загрузки в обработку нажатием кнопки "Загрузить". При загрузке автоматически устанавливается соответствие данных файла данным в базе. Соответствие происходит путём определения лицевого счета и льготника (жилец, имеющий право на получение субсидий и льгот, и относящийся к текущему лицевому счету). Если в строке таблицы загрузки лицевой счет или льготника в автоматическом режиме определить не удалось (о чём будет выдано сообщение), то такая строка будет подсвечена серым цветом. Пользователю предоставляется возможность установить соответствие вручную, выбрав в "серой" строке соответвтсующий лицевой счет в колонке "Лицевой счет в базе" или льготника в колонке "Льготник".
Примечание: Строки с неопределенным лицевым счётом или льготником не участвуют в механизме загрузки – данные, о начисленных по ним суммам льгот, загружены в базу не будут!

 

После загрузки данных из файла можно приступать к загрузке данных в базу. Для этого можно просто нажать кнопку "Загрузить". При этом на основании данных таблицы загрузки будет сформирован и записан новый документ "Начисление услуг в сторонней программе". Также, пользователю предоставляется возможность производить загрузку данных в уже существующий документ "Начисление услуг в сторонней программе". Для этого в поле "Начисление услуг" необходимо выбрать не проведенный документ "Начисление услуг в сторонней программе" и нажать кнопку "Загрузить". При этом необходимо выбрать вариант записи данных в документ:

  • "Перезаписать" - выбранный документ будет полностью перезаписан данными из выгрузки;
  • "Дополнить" - в табличную часть выбранного документа будут добавлены данные из таблицы загрузки (шапка, предыдущие данные в табличной части и подвал документа останутся без изменений);
  • "Пропустить" - отменить операцию записи.

Проведение созданного документа "Начисление услуг в сторонней программе" предоставляется пользователю. При проведении в базе формируются записи о начисленных в ОСЗН льготах в разрезе услуг. Примечание: при формировании документа суммы начисленных льгот только по тем услугам, которые назначены на соответствующий лицевой счет и которым на форме настроек выгрузки установлено соответствие с услугами в ОСЗН.
При этом, если в настройках "Учетной политики (ЖКХ)" на вкладке "Льготы" установлен флаг "Не учитывать льготы во взаиморасчетах", то данные о начисленных льготах будут отображаться в квитанциях чисто информативно. Если же этот флаг не установлен, то загруженные суммы льгот будут добавляться (вычитаться) к текущим начислениям.
