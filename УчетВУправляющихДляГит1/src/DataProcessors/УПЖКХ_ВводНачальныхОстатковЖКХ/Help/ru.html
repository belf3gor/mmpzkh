Ввод начальных остатков по данным ЖКХ

Обработка предназначена для ввода входящих остатков в информационную базу на основе данных документов ЖКХ «Ввод начального сальдо».

Для ввода остатков необходимо указать организацию и дату ввода начальных остатков, документы ввода начальных остатков будут формироваться в программе на указанную дату.

Ввод начальных остатков производится в несколько этапов:

  • «Заполнение документов». На данном этапе производится заполнение списка документов ЖКХ «Ввод начального сальдо», по которым будут формироваться входящие остатки. Список документов можно заполнить с помощью кнопок командной панели:
    • Добавить документ остатков - выбирается документ из списка документов «Ввод начального сальдо» по организации;
    • Заполнить список документов - табличная часть автоматически заполнится проведенными документами «Ввод начального сальдо» по организации;
    Переход к следующему этапу «Данные по остаткам» выполняется по кнопке «Далее»;
  • «Данные по остаткам». На основе документов ввода начального сальдо, указанных на предыдущем этапе, производится автоматическое заполнение остатков в разрезе контрагентов, договоров и документов расчетов. В строках табличной части счет учета заполняется автоматически согласно регистру сведений «Счета учета расчетов с контрагентами». При необходимости, данные по остаткам, сформированные автоматически, можно отредактировать вручную.

    Порядок заполнения граф:

    • Счет учета выбирается из ограниченного списка плана счетов бухгалтерского учета. Для выбора доступны счета групп 62 «Расчеты с покупателями и заказчиками» и 76 «Расчеты с разными дебиторами и кредиторами»;
    • Документ расчетов доступен для заполнения только в тех строках табличной части, в которых для счета учета ведётся аналитический учет по документам расчетов.

    На данном этапе возможен переход к предыдущему этапу «Заполнение документов» по кнопке «Назад». В случае, если данные по остаткам сформированы и производится изменение списка документов «Ввод начального сальдо» на этапе «Заполнение документов», то будет задан вопрос: следует ли очищать данные по остаткам. При утвердительном ответе программа очистит данные по остаткам и затем перезаполнит их автоматически при возврате на этап «Данные по остаткам», в противном случае - обработка не позволит вносить изменения в список документов начального сальдо.

    Переход к следующему этапу «Завершение» выполняется по кнопке «Далее». Если при переходе к следующему этапу не указаны документы расчетов для счетов учета, на которых ведется аналитический учет по документам расчетов, то обработка предложит заполнить их автоматически. При этом будут созданы документы «Документ расчетов с контрагентами (ручной учет)» по текущей организации на дату ввода остатков.

    Примечание
    Переход к следующему этапу не возможен в случаях, если в строках табличной части не указаны счета учета либо документы расчетов для счетов учета, на которых ведется аналитический учет по документам расчетов.

  • «Завершение». На данном этапе производится автоматическое формирование документов «Ввод начальных остатков». Документы ввода начальных остатков формируются по следующим разделам:
    • «Расчеты с покупателями и заказчиками (счет 62)»;
    • «Расчеты с разными дебиторами и кредиторами (счет 76)».

    После окончания формирования входящих остатков по данным ЖКХ рекомендуется произвести контроль введенных данных.

    По кнопке «Закрыть» закрывается форма обработки.
