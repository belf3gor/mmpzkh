Объекты учета
Если установлен флажок Автоматически пересчитывать общие площади зданий то общая площадь всех жилых и нежилых помещений, а также помещений, входящих в состав общего имущества, будет пересчитываться автоматически при изменении площадей помещений.

Учет жильцов

Если в базе не ведется учет проживающих и зарегистрированных жильцов, то необходимо установить флажок Указывать количество проживающих и зарегистрированных готовыми числами. После установки флажка в качестве количества проживающих и количества зарегистрированных программа будет использовать готовые числа, введенные в документах "Открытие лицевого счета" и "Изменение лицевого счета". В противном случае данные значения будут рассчитываться согласно заполненному списку жильцов.

Учет собственников

Можно задавать параметры учета собственников.

Если установить флажок Использовать упрощенный вариант учета собственников помещений, то вводить данные о собственности можно будет с помощью документов "Открытие лицевого счета" и "Изменение лицевого счета" одновременно с изменением данных о жильцах.

Если данный флажок не будет установлен, то данные о собственности можно будет регистрировать отдельным документом "Установка собственников помещений". При этом данные по собственности в документах "Открытие лицевого счета" и "Изменение лицевого счета" будут отображаться справочно.зависимость услуг.

Учет начислений

Задаются параметры округления объемов начисления услуг.

Настройка точности округления используется при расчете объемов начисления услуг и указывается в виде числа, которое определяет количество знаков после запятой.

Зависимые услуги:

  • Задается настройка установки зависимости услуг в разрезе зданий.

    Если установлен флажок Указывать зависимости услуг в разрезе зданий, то при установке зависимостей услуг появляется возможность указать здание, для которого данная зависимость действует. Если настройка используется, а в зависимости услуг здание не указано, то данная зависимость будет действовать на все здания, для которых индивидуально не введена зависимость услуг.

  • Выбирается способ начисления зависимых услуг в случае, когда хотя бы по одной из услуг-оснований нет действующего прибора учета.

    Если выбран вариант В соответствии с объемами начисления услуг-оснований, то начисление зависимых услуг выполняется в соответствии с объемами начисления услуг-оснований, независимо от наличия установленных индивидуальных приборов учета.

    Если выбран вариант По собственному нормативу зависимой услуги, то если хотя бы по одной из услуг-оснований зависимой услуги нет установленного индивидуального прибора учета, начисление производится по собственному нормативу зависимой услуги.

Добровольное страхование:

Если установлен флажок Вести учет начислений по добровольному страхованию, то в программе будет доступен функционал учета начислений по услуге добровольного страхования имущества собственников помещений.

При установке данного флага становится доступным поле Услуга добровольного страхования, предназначенное для выбора ссылки на услугу, которая является услугой добровольного страхования. При этом:

  • Назначение и установка тарифа по данной услуге выполняется стандартным способом;
  • Начислить услугу можно документами Начисление услуг с видом операции "Начисление добровольного страхования", Разовое начисление услуг с видом операции "Прочие начисления", Ввод фактических затрат на объект;
  • По добровольному страхованию нельзя начислять пени, предоставлять рассрочку, начислять льготы, вводить начальное сальдо, корректировать начисления;
  • Оплата добровольного страхования фиксируется документом Регистрация оплаты. При этом оплатить добровольное страхование можно только тогда, когда оплачены все долги и текущие начисления;
  • Сведения о начислениях по добровольному страхованию будут выводиться в платежных документах отдельной - необязательной к оплате - суммой.

Корректировки начислений

Задается вариант изменения задолженности в корректировках начислений.

Если установлен вариант изменения расчетного периода, за который произведена корректировка, то корректировка начислений изменяет задолженности по взаиморасчетам того расчетного периода, за который производится корректировка.

Например:
Если 10 октября введена корректировка начислений за март, то изменяется задолженность марта месяца.

Если установлен вариант изменения текущего расчетного периода, то корректировка начислений изменяет задолженности по взаиморасчетам текущего расчетного периода.

Например:
Если 10 октября введена корректировка начислений за март, то изменяется задолженность октября месяца.

Корректировка начислений может изменять задолженность того или иного расчетного периода в зависимости от варианта изменения задолженности и влияет на следующие разделы учета:

  • Начисление пени - сумма корректировки будет учитываться при начислении пени в зависимости от расчетного периода, к которому она отнесена в соответствии с текущей настройкой корректировки. Например, если корректировка с доначислением отнесена к текущему расчетному периоду, то отсчет даты начала начисления пени на эту корректировку будет производиться от текущего расчетного периода.
  • Регистрация оплаты - при распределении оплат в регистрации оплаты сумма корректировки будет относиться к тому периоду задолженности или переплаты, к которому она отнесена в соответствии с текущей настройкой корректировки.

Приборы учета

Повышающие коэффициенты:

Если установлено использование повышающих коэффициентов, то норматив потребления коммунальной услуги в жилых помещениях определяется с учетом повышающего коэффициента согласно Постановлению №344 Правительства РФ. Повышающий коэффициент учитывается при наличии технической возможности установки коллективных (общедомовых), индивидуальных или общих (квартирных) приборов учета коммунальных услуг. В форме доступен переход к настройкам сведений о возможности установки приборов учета в зданиях и значений повышающих коэффициентов. При этом:

  • если в здании присутствует возможность установки прибора учета услуги, но в помещении прибор учета не установлен, то начисление услуги по нормативу потребления производится с учетом повышающего коэффициента, заданного для услуги;
  • если в здании отсутствует возможность установки прибора учета услуги или в помещении установлен прибор учета, то повышающий коэффициент при начислении услуги не используется.

 

Выбирается вариант расчета среднего расхода по приборам учета.

Для расчета среднего расхода брать месяцы, предшествующие:

  • Расчетному месяцу - в таком случае для расчета берутся показания введенные за период месяцев, определенный в настройке услуги "Производить по среднему значению при отсутствии показаний за период", например 6 месяцев, предшествущих месяцу расчета.

Для данного варианта задается способ определения среднего расхода по приборам учета.

Способ определения среднего расхода по приборам учета может принимать следующие значения:

  • Отношение расхода за период к количеству месяцев с ненулевым расходом - средний расход по приборам учета для расчета начислений по среднему определяется как отношение расхода за период к количеству месяцев с ненулевым расходом в данном периоде;

В количество месяцев с ненулевым расходом включаются месяца, в которых было начисление услуги по показаниям приборов учета.

  • Отношение расхода за период к количеству месяцев с действующим ПУ - средний расход по приборам учета для расчета начислений по среднему определяется как отношение расхода за период к количеству месяцев с действующим прибором учета в данном периоде.

В количество месяцев с действующим прибором учета включаются месяца, в которых действовал или был установлен хотя бы один прибор учета.

В качестве расхода за период берется объем начислений по показаниям приборов учета, введенным за период расчета среднего расхода. Начисления по среднему расходу и по нормативу не учитываются при расчете среднего расхода приборов учета.

  • Месяцу ввода последних показаний - в таком случае для расчета берутся показания введенные за период месяцев, определенный в настройке услуги "Производить по среднему значению при отсутствии показаний за период", например 6 месяцев, предшествующих месяцу ввода последних показаний по прибору учета, с учетом самого месяца ввода последних показаний.

Пример:
В настройках услуги указано "Производить расчет по среднему значению" при отсутствии показаний за 6 месяцев.

Последние показания были введены в октябре месяце. В таком случае для расчета среднего будут взяты показания с мая по октябрь.

Задаются настройки способа распределения показаний коллективных приборов учета, установленных на несколько помещений / лицевых счетов, и приборов учета, установленных на несколько подъездов / зданий.

Способ распределения показаний приборов учета, установленных на несколько помещений / лицевых счетов может принимать следующие значения:

  • Пропорционально зарегистрированным - распределение показаний прибора учета между лицевыми счетами, на которые он установлен, будет производиться пропорционально количеству зарегистрированных человек;
  • Пропорционально проживающим - распределение показаний прибора учета между лицевыми счетами, на которые он установлен, будет производиться пропорционально количеству проживающих человек;
  • Пропорционально долям лицевых счетов - распределение показаний прибора учета между лицевыми счетами, на которые он установлен, будет производиться пропорционально долям собственности лицевых счетов.

Способ распределения показаний приборов учета, установленных на несколько подъездов / зданий может принимать следующие значения:

  • Поровну - распределение показаний прибора учета между подъездами / зданиями, на которые он установлен, будет производиться поровну;
  • Пропорционально площадям помещений - распределение показаний прибора учета между подъездами / зданиями, на которые он установлен, будет производиться пропорционально виду площади помещений (квартир и офисов), указанному в настройке Вид площади.

 

Задаются настройки сторнирования индивидуальных начислений.

Можно выбирать, по какому тарифу производить сторнирование начислений:

  • по тарифу, действовавшему на момент начисления - при сторнировании начислений по услуге используется тариф, действовавший в периоде начисления при отсутствии показаний индивидуального прибора учета;
  • по актуальному тарифу - сторнирование начислений по услуге производится по актуальному на момент сторнирования тарифу. 

Если установлен флажок Сторнировать начисления только по действующим приборам учета, то сторнирование начислений производится только с момента установки текущего действующего счетчика. Если настройка не используется, то сторнирование начислений производится за весь период вне зависимости от установок приборов учета.

Если установлен флажок Учитывать сторнирование индивидуальных начислений при расчете ОДН, то при сторнировании начислений, произведенных при отсутствии показаний ИПУ, объем расхождений между начислениями по показаниям ИПУ и начислениями при отсутствии показаний будет учитываться при расчете ОДН. Если флажок не установлен, то при расчете ОДН сторнирование начислений учитываться не будет. При установленном флажке становится доступной настройка Применяется с, где необходимо указать месяц, с которого будет применяться учет, а также настройка выбора варианта учета сторнированных начислений при расчете ОДН, которая может принимать следующие значения:

  • В пределах объема текущего начисления - объем сторнируемых начислений, произведенных при отсутствии показаний ИПУ, учитывается ежемесячно в объеме, не превышающем объем текущих начислений. При этом остаток неучтенного в текущем месяце объема сторно будет учтен в последующих месяцах.
  • В полном объеме - объем сторнируемых начислений, произведенных при отсутствии показаний ИПУ, учитывается сразу в полном объеме в месяце сторнирования начислений.

Пример:
Имеется услуга "Горячее водоснабжение", по которой установлен общедомовой прибор учета. Расход общедомового прибора учета в январе составил 100 м3.

Расчет по лицевому счету №00000001 выполняется по показаниям индивидуального прибора учета.

Пусть в январе по данному л/с не были переданы показания. В результате в январе было произведено индивидуальное начисление по нормативу в объеме равном 8 м3.

Пусть суммарный объем потребления по другим квартирам составил 90 м3.

Произведено распределение ОДН в объеме 100-90-8 = 2 м3, т.е. 8 м3 по л/с были учтены при распределении ОДН.

В феврале месяце по л/с были переданы показания в размере 3 м3.

При этом в феврале произведено индивидуальное начисление по показаниям в объеме равном 3 м3, а также автоматическое сторнирование норматива в объеме 8 м3, начисленных в январе.

Рассмотрим, каким образом будет произведен расчет ОДН в феврале в зависимости от выбранного варианта учета сторнированных начислений (для простоты предположим, что расход общедомового прибора учета в феврале и марте также составил 100 м3, а суммарный объем потребления по другим квартирам составил 90 м3):

1) В пределах объема текущего начисления

При определении объема ОДН к распределению по данному л/с будет учитываться весь индивидуальный объем и объем сторно, в размере, не превышающем индивидуальный объем, а именно, 3 - минимум(8 и 3) = 0 м3 (учетный индивидуальный объем).

В феврале будет распределен объем ОДН в размере 100-90=10 м3. При этом остаток неучтенного объема сторно в размере 8 - 3 = 5 м3 будет учтен в следующих периодах начисления.

Если предположить, что в марте индивидуальный расход по показаниям прибора учета по лицевому счету составил 7 м3, тогда объем ОДН в марте будет рассчитан как 100-90-(7-минимум(7 и 5)) = 8 м3.

Таким образом, объем сторнируемых начислений, произведенных при отсутствии показаний ИПУ, будет учтен в феврале в размере 3 м3 и в марте 5 м3.

2) В полном объеме

При определении объема ОДН к распределению по данному л/с будет учитываться весь индивидуальный объем и весь объем сторно, а именно 3 м3 – 8 м3 = -5м3 (учетный индивидуальный объем).

В феврале будет распределен объем ОДН в размере 100-90-(-5)=15 м3.

На март остатки уже не переносятся, т.к. объем сторнируемых начислений, произведенных при отсутствии показаний ИПУ, был учтен полностью в феврале.

Расчет ОДН в марте составит 100-90-7=3 м3.

Для обоих вариантов суммарный размер ОДН за 3 месяца составит 20 м3.

Контроль показаний приборов учета:

Если установлено использование контроля показаний приборов учета, то при вводе показаний счетчиков выполняется проверка превышения введенного расхода над средним расходом по счетчику за последние 6 месяцев, а также проверка отрицательного расхода. Размер превышения в процентах указывается дополнительно.

Пример:
Имеется услуга "Электроснабжение", по которой установлен прибор учета.

Размер допустимого превышения установлен в значении 50%.

Расход по прибору учета в январе составил 100 кВт.

Средний расход за последние 6 месяцев составил 10 кВт.

В таком случае считается, что введенный расход некорректен, тогда в документе ввода показаний необходимо либо подтвердить корректность введенного расхода, либо изменить его значение.

Расчет льгот

Задаются параметры расчета льгот.

Если установить флажок Использовать новый алгоритм расчета льгот, то при расчете льгот будет использоваться новый алгоритм. В новом алгоритме расчет льгот производится не для льготной категории, а для каждого льготника. При этом учитываются периоды изменения параметров для расчета льгот, таких как:
• даты установки и прекращения действия льготы;
• даты назначения и приостановки услуг;
• тарифы услуг;
• размеры нормативов, ограничивающих объем предоставляемой услуги;
• количество проживающих, зарегистрированных жильцов / близких родственников, на которых распространяется скидка;
• площади помещений.
Расчет всегда выполняется по наиболее выгодной сумме льготы на каждом подпериоде расчета.

Документы расчета льгот, введенные до перехода на новый алгоритм расчета льгот будут рассчитываться по старому алгоритму расчета.

Регистрация оплаты

Операции по кассе:

На данной закладке можно задавать способ печати чеков.

Если переключатель установлен Из документов "Приходный кассовый ордер"/"Расходный кассовый ордер", то для печати чеков из документа "Регистрация оплаты" с видом операции "Поступление в кассу"/"Возврат оплаты из кассы" в табличной части обязательно должен быть указан документ оплаты - "Приходный кассовый ордер"/"Расходный кассовый ордер" соответственно.

Если переключатель установлен Из документа "Регистрация оплаты", то для печати чеков из документа "Регистрация оплаты" с видами операций "Поступление в кассу", "Возврат оплаты из кассы" в табличной части не обязательно указывать документ оплаты. Также при выборе данного способа печати становятся видимыми закладки "Операции по банку" и "Настройка печати чеков", в которых устанавливаются дополнительные параметры печати чеков.

Операции по банку:

На данной закладке можно установить флажок Разрешить печать чеков при оплате банковской картой. Если данный флажок установлен, то для документа "Регистрация оплаты" с видами операций "Поступление на банковский счет", "Возврат оплаты с р/с" становится доступной печать кассовых чеков.

Это может быть необходимым в случае оплаты услуг пластиковой картой, при этом собственнику выдается кассовый чек с отметкой о получении безналичного платежа.

Настройка печати чеков:

На данной закладке можно установить флажок Печатать чеки с расшифровкой по услугам. Если установить данный флажок, то при печати чека из документа "Регистрация оплаты" будет выполняться печать чеков с расшифровкой по услугам. В противном случае печать чеков будет выводиться на общую сумму (без расшифровки по услугам).
