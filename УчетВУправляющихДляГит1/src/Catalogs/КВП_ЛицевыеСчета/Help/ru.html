Справочник "Лицевые счета"

Данный справочник используется для хранения списка лицевых счетов, по которым производится начисление жилищно–коммунальных услуг. Справочник редактируется из документов «Открытие лицевого счета», «Изменение лицевого счета» и «Закрытие лицевого счета».

Форма списка элементов

На командной панели диалогового окна списка лицевых счетов расположены следующие кнопки:

  • «Создать на основании» - по нажатию кнопки выводится меню кнопок: «Открытие лицевого счета», «Изменение лицевого счета» и «Закрытие лицевого счета». При нажатии кнопок открывается документ, соответствующий наименованию кнопки.
  • «Печать» - по нажатию кнопки выводится меню с кнопкой «Действующие начисления». При нажатии кнопки открывается отчет, соответствующий наименованию кнопки.
  • «Создать» - по нажатию кнопки выводится меню кнопок: «Регистрация оплаты», «Корректировка начислений». При нажатии кнопок открывается документ, соответствующий наименованию кнопки.
  • «Отчеты» - по нажатию кнопки выводится меню кнопок: «Краткий список проживающих по лицевому счету», «Начисления и задолженность», «Действующие начисления», «Квитанции и извещения». При нажатии кнопок открывается отчет, соответствующий наименованию кнопки.
  • «Заполнить нормы потребления» - по нажатию кнопки открывается обработка «Заполнение норм потребления услуг».

    Внизу диалогового окна списка лицевых счетов расположено поле, состоящее из закладок:

    • «Жильцы» - выводится список проживающих по текущему лицевому счету;
    • «Основная информация» -  выводится общая информация по лицевому счету, такая как сведения о собственности, площадь помещения и пр.
    • «Сведения для взаиморасчетов» - выводится информация о сведениях для взаиморасчетов по лицевому счету, используемых при отражении в регламентированном учете. Информацию можно отобрать по организации, а также включить вывод истории изменения сведений для взаиморасчетов.
    • «Взаиморасчеты» - выводится общая информация по задолженности, начислениям и оплатам по текущему лицевому счету. Также выводятся графики, сведения которых содержат следующие данные: начисления и оплаты, процент собираемости оплат, начисления и оплаты пени, процент собираемости оплат по пени, задолженность по месяцам.  Общая информация  и графики формируются за два месяца: выбранный месяц в поле "Период" и за предыдущий ему месяц. Отбор сведений по организации в общей информации и графиках осуществляется по выбранной организации в поле «Организация». Изменить вид графиков можно в окне настроек, открывающемся при нажатии кнопки «Настройки графиков».
    • «Действующие начисления» - выводится список действующих начислений по услугам на рабочую дату по текущему лицевому счету;
    • «Карточка расчетов» - выводится сведения о расчетах по текущему лицевому счету за период, выбранный в поле «Период». Также выводятся графики, которые формируются за данный выбранный период. Сведения графиков содержат следующие данные: начисления и оплаты, процент собираемости оплат, начисления и оплаты пени, процент собираемости оплат по пени, задолженность по месяцам. Изменить вид графиков можно в окне настроек, открывающемся при нажатии кнопки «Настройки графиков». Отбор сведений по организации в таблице сведений о расчетах и графиках осуществляется по выбранной организации в поле «Организация».
    • «Журнал документов» - выводится список документов по текущему лицевому счету;
    • «Счетчики» - выводится список действующих и отключенных на рабочую дату приборов учета, закрепленных за текущим лицевым счетом. Для настройки отображения только действующих или всех, включая отключенных, приборов учета текущего лицевого счета добавлена кнопка "Показать отключенные счетчики". Также выводятся графики, сведения которых содержат данные о расходах по приборам учета текущего лицевого счета. Графики формируются за 6 последних месяцев. При переходе на строки списка действующих приборов учета, закрепленных за текущим лицевым счетом, график расхода по текущей услуге прибора учета выводится первым. Изменить вид и параметры графиков можно в окне настроек, открывающемся при нажатии кнопки «Настройки графиков».
    • «Примечания» - выводится список примечаний к текущему лицевому счету.
    • «Льготники» - выводится список льготников по текущему лицевому счету. Также выводятся графики, сведения которых содержат данные о начислениях и начисленных льготах по текущему лицевому счету либо группе лицевых счетов. Графики формируются за 6 последних месяцев. Изменить вид графиков можно в окне настроек, открывающемся при нажатии кнопки «Настройки графиков». Также имеется возможность настраивать перечень данных, которые будут выводиться на форме: либо только список льготников, либо только графики, либо список льготников и графики. Данная возможность представляет собой набор кнопок-переключателей с соответствующими наименованиями. Отбор сведений по организации в общей информации и графиках осуществляется по выбранной организации в поле «Организация».

    Форма редактирования элемента

    Ввод нового элемента производится из документов через диалог ввода элемента справочника. Форма элемента справочника «Лицевые счета» состоит из пяти закладок: «Общие данные», «Данные по документам», «Нормы потребления услуг», «Комнаты», «Сведения о собственнике».

    Закладка «Общие данные»

    На этой закладке расположены следующие элементы:

    • «Группа» – указывается наименование группы, которой принадлежит данный лицевой счет;
    • «Наименование» – наименование лицевого счета;
    • «Идентификатор» – указывается номер или набор символов, по которому можно точно идентифицировать этот лицевой счет;
    • «Код» – код элемента справочника (устанавливается автоматически);
    • «Адрес» – выводится ссылка на помещение, на которое открыт лицевой счет;
    • «Адрес доставки счетов и квитанций» – указывается адрес доставки счетов и квитанций, если адрес лицевого счета по каким-либо причинам не может быть использован в качестве адреса доставки;
    • «Печать квитанций на бумажных носителях» – выбирается "Печатать", если необходимо выводить квитанции на бумажном носителе, и "Не печатать", если достаточно получить электронную форму квитанции. Используется для подбора объектов при формировании квитанций;
    • «Количество проживающих» – выводится количество проживающих по лицевому счету;
    • «Количество зарегистрированных» – выводится количество зарегистрированных по лицевому счету;
    • «Дата открытия» – выводится дата открытия лицевого счета;
    • «Дата закрытия» – выводится дата закрытия лицевого счета (если лицевой счет не закрыт, поле остается пустым);
    • «История изменения» - ссылка для перехода к истории изменения сведений для взаиморасчетов по текущему лицевому счету;
    • «Организация» – выводится организация, с которой заключен договор (информация выводится только в случае, если сведения для взаиморасчетов заданы по одной организации);
    • «Контрагент» – выводится контрагент, который будет подставляться в качестве субконто при формировании проводок по начислению (информация выводится только в случае, если сведения для взаиморасчетов заданы по одной организации);
    • «Договор контрагента» - выводится договор контрагента, на основе которого производится обслуживание лицевого счета (информация выводится только в случае, если сведения для взаиморасчетов заданы по одной организации);
    • «Вид собственности» – выводится вид частной собственности (совместная или долевая);
    • «Доля собственности Л/С» – указывается доля собственности текущего лицевого счета в указанном помещении;
    • «Ответственный собственник, наниматель» – выводится проживающий, отображаемый в качестве владельца; заполняется автоматически при добавлении проживающего, являющегося квартиросъемщиком.

    В коде и идентификаторе лицевого счета при необходимости печати штрих кодов в квитанциях не рекомендуется использование кириллицы. 

    Закладка «Контакты»

    На этой закладке расположены следующие элементы:

    • «Телефон» – указывается контактный телефон лицевого счета;
    • «Email» – указывается адрес электронной почты лицевого счета;
    • «Телефоны, email собственников и жильцов» – таблица в которой отображается телефон, email собственнников и жильцов, относящихся к лицевому счету.
      Информация выводится по стобцам:
      • ФИО/Наименование - выводится ФИО если собственник/жилец является физическим лицом или наименование контрагента, если собственник является контрагентом. Здесь указываются собственники, а также зарегистрированные/проживающие по лицевому счету. В скобках указывается категория: отв. собственник, собственник, жилец. Представляет собой гиперссылку, позволяющую переходить на карточку физического лица/контрагента для редактирования контактной информации.
      • Телефоны - список телефонов собственника/жильца.
      • Email - список адресов электронной почты собственника/жильца. Представляет собой гиперссылку, нажатие на которую формирует новое электронное письмо.

    Закладка «Данные по документам»

    На этой закладке расположены следующие поля:

    • «Дата договора о приватизации» – указывается дата договора о приватизации жилой площади;
    • «Дата выдачи документа на жилье» – указывается дата выдачи документов на жилье;
    • «Документ–основание» – указывается документ открытия лицевого счета;
    • «Номер договора о приватизации» – указывается номер договора о приватизации;
    • «Номер ордера или др. док.» – указывается номер ордера или другого документа на жилую площадь;

    Закладка «Нормы потребления услуг»

    На этой закладке указываются нормы потребления услуг. На лицевой счет можно установить только те нормы потребления, в настройках которых указано, что норма устанавливается «На лицевой счет», а в основании нормы не используется вид жильцов «Близкий родственник».

    Закладка «Комнаты»

    На этой закладке выводится список комнат, закрепленных за лицевым счетом.

    Закладка «Сведения о собственнике»

    На этой закладке расположены следующие реквизиты:

    • группа «Данные для доступа на сайт». Если организация осуществляет обслуживание лицевых счетов с помощью интернет-порталов, то в поле "Логин для доступа" выводится логин для доступа в личный кабинет.
    • группа «Данные по документу, удостоверяющему личность». В этой группе выводятся данные о документе, удостоверяющем личность собственника:
      • Вид документа;
      • Серия документа;
      • Номер документа;
      • Дата выдачи документа;
      • Кем выдан документ;
      • Код подразделения;

    На верхней командной панели расположены кнопки:

    • «Записать и закрыть» – по нажатию на кнопку данные справочника сохраняются и форма закрывается;
    • «Записать» – по нажатию на кнопку данные справочника сохраняются, форма не закрывается.

    Справочник можно открыть из «Управление лицевыми счетами / Лицевые счета».

    Закладка «Сведения для ГИС ЖКХ»

    На этой закладке хранятся:

    • «Единый лицевой счет (ЕЛС)» – номер единого лицевого счета в ГИС ЖКХ;
    • «Идентификатор ЖКУ» – идентификатор ЖКУ, присвоенный лицевому счету на стороне ГИС ЖКХ при экспорте лицевых счетов из информационной базы в ГИС ЖКХ. Данный идентификатор используется для идентификации лицевого счета при выгрузке платежных документов.

     

    Блок «Идентификаторы лицевых счетов капитального ремонта»:

    • «Единый лицевой счет КР (ЕЛС КР)» – номер единого лицевого счета в ГИС ЖКХ с типом "Капитальный ремонт" (ЛС КР);
    • «Идентификатор ЖКУ» – идентификатор ЖКУ КР лицевого счета в ГИС ЖКХ с типом "Капитальный ремонт" (ЛС КР).

    В случае, когда организация ведет раздельный учет услуг капитального ремонта (КР), и печатает отдельные платежные документы по капитальному ремонту, необходимо заполнить идентификаторы лицевых счетов капитального ремонта:
    - Если лицевой счет создан только для начисления услуг капитального ремонта, то выгрузку в ГИС лицевых счетов необходимо выполнять с установленной настройкой "Выгружать лицевые счета капитального ремонта", а загрузку идентификаторов по таким лицевым счетам необходимо выполнять с установленной настройкой "Загружать идентификаторы лицевых счетов капитального ремонта". При этом при загрузке идентификаторов будут заполняться идентификаторы ЕЛС КР и ЖКУ КР.

    - Если для начисления услуг капитального ремонта отдельные лицевые счета не создаются, но при этом печатаются отдельные платежные документы по КР, то выгрузку таких лицевых счетов, а также загрузку идентификаторов таких лицевых счетов необходимо выполнить дважды:
    1) Выполнить выгрузку лицевых счетов с установленной настройкой "Выгружать лицевые счета капитального ремонта", а затем загрузить с портала ГИС идентификаторы лицевых счетов КР, выбрав настройку "Загружать идентификаторы лицевых счетов капитального ремонта". При этом при загрузке идентификаторов будут заполняться идентификаторы ЕЛС КР и ЖКУ КР.
    2) Выполнить выгрузку лицевых счетов с установленной настройкой "Выгружать лицевые счета управляющей организации", а затем загрузить с портала ГИС идентификаторы лицевых счетов УО, выбрав настройку "Загружать идентификаторы лицевых счетов управляющей организации". При этом при загрузке идентификаторов будут заполняться идентификаторы ЕЛС и ЖКУ.
    Таким образом в ГИС ЖКХ будут созданы 2 лицевых счета с разными типами (ЛС УО и ЛС КР), но с одинаковым наименованием.
