Маршрут подписания

Справочник используется для формирования наборов требований к подписанию электронных документов. В качестве требования указывается пользователь, который должен осуществлять подписание документа. Его можно уточнить указанием сертификата электронной цифровой подписи, который должен быть использован при подписании.
Требования могут объединяться в логические условия с произвольным уровнем вложенности.

Настройка состава маршрута

Состав маршрута подписания задается в виде дерева. Каждая ветвь дерева (требование) может быть двух видов:

  • Подписант - пользователь, который может подписать документ. Можно также указать конкретный сертификат электронно-цифровой подписи. В этом случае подписать документ можно будет только этим сертификатом. Используется в случаях, когда один пользователь использует несколько сертификатов для разных целей. Чтобы добавить требование данного вида, нужно нажать кнопку Добавить подписанта.
  • Группа - позволяет объединять подписантов в группы (логические условия). Может содержать другие группы либо подписантов. Чтобы добавить такое требование, нужно воспользоваться меню Добавить группу и выбрать конкретный вид.

В справочнике можно использовать группы (логические условия) трех видов:

  • Поставить любую из подписей
  • Поставить все подписи
  • Поставить все подписи по порядку
