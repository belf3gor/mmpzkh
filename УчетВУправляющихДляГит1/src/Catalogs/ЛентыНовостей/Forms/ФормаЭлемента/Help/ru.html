Редактирование ленты новостей

Основные данные

  • Наименование - произвольное наименование ленты новостей, видимое пользователям
  • Код - код ленты новостей, используемый подсистемой получения новостей
  • Обязательность - возможность отключения ленты новостей пользователем. Обязательные ленты новостей пользователь не сможет отключить и будет получать по ним новости всегда.
  • Видимость - здесь задаются исключения видимости ленты новостей. Лента новостей может быть по умолчанию видима всем, кроме некоторого списка пользователей. И наоборот - лента новостей по-умолчанию может быть от всех скрыта и видна только определенным пользователям.
  • Комментарий - произвольный текст

Закладка "Настройки загрузки новостей"

  • Это локальная лента новостей - признак, что лента новостей создана вручную, и заполняться она будет в самой информационной базе (вручную, пользователями с ролью РедакторНовостей или программно)
  • Протокол - протокол связи (для случая, если лента новостей не локальная). Возможные значения: http, https, file
  • Сайт - имя сайта, с которого будет производиться чтение новостей в формате atom или rss. Для случая, если лента новостей не локальная и протокол не file
  • Имя файла - имя файла на сайте (или в операционной системе), откуда будут регулярно считываться новости. Для случая, если лента новостей не локальная.
  • URL - результирующая ссылка на файл с новостями (результат сложения Протокол, Сайт и Имя файла). Если перейти по этой ссылке, то можно посмотреть в браузере самые актуальные новости.
  • Подключение - Вариант авторизации. Обычно все ленты новостей открыты для всех. Но в некоторых случаях придется ввести логин и пароль. Это настраивается в этом пункте. Возможные значения:
    • Без логина / пароля - логин и пароль для загрузки новостей не требуется (это вариант по-умолчанию)
    • Введено вручную (одинаковый для всех) - логин и пароль необходимо ввести вручную (он будет одинаков для всех областей данных)
    • Введено вручную (разный для всех) - логин и пароль необходимо ввести вручную (он будет отличаться для каждой области данных)
    • Значения логина / пароля из Веб-ИТС - значения логина и пароля будут взяты из настроек доступа к интернет-поддержке пользователей
  • Логин - логин для доступа к ленте новостей. для случая, когда Подключение - Введено вручную
  • Пароль - пароль для доступа к ленте новостей. для случая, когда Подключение - Введено вручную
  • Загрузка бинарных данных - можно выключить для уменьшения трафика. Настоятельно не рекомендуется это делать, т.к. в противном случае перестанут загружаться картинки, вложенные в новости

Закладка "Частота обновления"

На этой закладке настраивается частота обновления ленты новостей из интернета (в случае, если это не локальная лента новостей).

Если выбрать Обновлять только вручную, то подсистема чтения новостей перестанет автоматически загружать новости в эту ленту новостей и придется обновлять ленту новостей только вручную, с помощью обработки Управление настройками и обновлением для новостей (для открытия требуются административные права)

Закладка "Настройки отборов"

Если лента новостей позволяет настраивать категории, то на этой закладке можно настроить, по каким категориям пользователи смогут настраивать отборы, а также установить значения отборов, которые пользователи не смогут изменить.

Для локальных лент новостей список категорий настраивается на этой закладке.

В лентах новостей, поставляемых фирмой 1С, список категорий настраивается фирмой 1С и его нельзя изменить.

Закладка "Редактирование новостей"

Эта закладка видна только в случае, когда лента новостей является локальной (т.е. когда новости не загружаются в нее автоматически, а создаются пользователями с ролью РедакторНовостей).

Здесь можно настроить список пользователей - редакторов новостей, которым разрешено редактировать новости в этой локальной ленте новостей.

Закладка "Статистика"

На этой закладке можно посмотреть общее количество новостей по этой ленте новостей, а также посмотреть дату самой свежей новости. Также можно сбросить сохраненную дату последней загрузки новостей, чтобы инициировать повторную загрузку всех новостей.
