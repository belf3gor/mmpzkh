Зарплатный проект

Под зарплатным проектом понимается договоренность организации с банком о перечислении денежных средств организации на лицевые счета сотрудников, открытые в этом банке (см. http://v8.1c.ru/edi/edi_app/35).

Если в организации действует один зарплатный проект, и использовать обмен электронными документами в рамках него не планируется, то добавлять его в справочник необходимости нет.

Добавление зарплатного проекта в этот справочник может потребоваться в двух случаях:
• если планируется с помощью встроенных в программу средств обмениваться с банком электронными документами в рамках этого проекта;
• если в организации действует несколько зарплатных проектов, даже если ни по одному из них встроенный обмен электронными документами использовать не планируется.

Карточка зарплатного проекта

Банк выбирается из соответствующего справочника, поэтому в нем необходимо создать банк. Важно указать корректный БИК – он используется при контроле корректности номеров лицевых счетов сотрудников.

Если по проекту планируется использовать электронный обмен, то следует установить соответствующий флажок и заполнить сведения о договоре и о банке. Эта информация будет использоваться при формировании файлов для банка.

После установки флажка становятся доступны рабочее место для обмена с банками по зарплатным проектам и соответствующие документы, которые формируются из этого рабочего места.

В программе реализован обмен по открытому универсальному стандарту электронного обмена информацией (см. http://v8.1c.ru/edi/edi_stnd/109/). Стандарт развивается, в связи с чем появляются его новые версии. Как правило, если банк поддерживает этот стандарт, то используется последняя его версия. При необходимости в поле Формат файла можно выбрать формат предыдущих версий – необходимую версию следует уточнить в банке. В некоторых случаях по требованию банка может потребоваться указать определенную кодировку, в которой будут формироваться файлы.

Внимание! Если в программе создается первый зарплатный проект, для которого указывается использование электронного обмена, то для отображения в его карточке полей для уточнения формата и кодировки его следует записать.

Название зарплатного проекта используется только внутри программы – для удобства выбора, если проектов несколько. Оно формируется автоматически, но может быть отредактировано.

В рамках зарплатного проекта банк может выпускать карточки одной или нескольких систем (на выбор сотрудника), например, VISA и MasterСard. Поддерживаемые банком системы следует перечислить в таблице – затем они будут доступны для выбора при оформлении заявки на открытие лицевых счетов.
