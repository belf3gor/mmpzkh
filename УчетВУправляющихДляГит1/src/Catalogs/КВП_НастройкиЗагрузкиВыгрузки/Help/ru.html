Справочник "Настройки загрузки/выгрузки"

Данный справочник используется для хранения настроек загрузки файла реестра платежей и показаний приборов учета.

На форме элемента справочника расположены следующие настройки:

  • «Наименование» - указывается наименование настроек загрузки (например, название используемой системы приема платежей);
  • «Код» - код элемента справочника;
  • «Вид загрузки/выгрузки» - из списка выбирается вид операции("Загрузка из систем приема платежей", "Загрузка из систем сбора показаний приборов учета", "Универсальная выгрузка данных");
  • «Вариант настройки формата реестра» - из списка выбирается вариант настройки формата реестра показаний приборов учета. Поле доступно для Вида загрузки/выгрузки "Загрузка из систем сбора показаний приборов учета". Доступны значения:
    • «Вручную» - выбор данного значения настройки "Вариант настройки формата реестра" указывает на то, что используется фиксированный порядок значений параметров "Прибор учета" в каждой строке реестра показаний. Например: в каждой строке реестра показаний на позиции ("Номер колонки") равной 10 хранится только дневное показание текущего Прибора учета 1.
    • «По шаблону» - выбор данного значения настройки "Вариант настройки формата реестра" указывает на то, что в качестве формата реестра показаний используется определенный формат файла со сведениями о показаниях приборов учета. Например, формат реестра показаний соответствует формату выгрузки "Вариант 8" в обработке "Выгрузка в Систему Город".
  • «Шаблон реестра показаний приборов учета» - указывается шаблон файла реестра показаний приборов учета. Поле доступно для Варианта настройки формата реестра "По шаблону".
  • «Кодировка» - с помощью переключателя выбирается кодировка загружаемого файла (DOS/Windows);
  • «Формат загружаемого файла» - из списка выбирается формат загружаемого файла.
  • «Вариант загрузки нулевых показаний» - настройка позволяет выбрать, каким образом нулевые показания в файле реестра будут загружены в документ "Ввод показаний счетчика". А именно:
    • Если выбрано "Загружать нулевые как 0", то в нулевые показания в документе "Ввод показаний счетчика" будут записаны как 0.
    • Если выбрано "Загружать нулевые как "Не введено", то в нулевые показания в документе "Ввод показаний счетчика" будут записаны как "Не введено".

Форматы загружаемого файла

  • «TXT, CSV» - имеют следующие настройки:
    • «Номер строки» - указывается номер строки, с которой необходимо начать загрузку;
    • «Признак итоговой строки» - указывается признак итоговой строки, начиная с которой не производить считывание строк (например, номер строки, символ или слово в итоговой строке);
    • «Символ-разделитель колонок» - указывается символ-разделитель колонок табличной части загружаемого файла (например, точка с запятой);
    • «Символ-разделитель дробной части» - указывается символ-разделитель дробной части числовых полей загружаемого файла (например, точка).
  • «XLS, DBF» - имеют следующие настройки:
    • «Номер строки» - указывается номер строки, с которой необходимо начать загрузку.
  • «XLS, DBF» - имеют следующие настройки:
    • «Тип атрибута» - определяет местоположение атрибутов в XML файле: либо внутри узла, либо отдельным подчиненным узлом.

    Табличная часть формы элемента справочника содержит колонки, которые соответствуют полям в загружаемом файле. Здесь необходимо указать соответствующие параметры.

    Вид операции«Загрузка из систем приема платежей»

    Список параметров для вида операции «Загрузка из систем приема платежей»:

    • «Дата» - дата приема платежа;
    • «Лицевой счет» - лицевой счет (определяется по коду, наименованию либо идентификатору элемента справочника «Лицевые счета» в зависимости от выбранного значения поля «Лицевой счет определяется по» );
    • «Сумма» - сумма платежа;
    • «Услуга» - услуга, по которой произведен платеж;
    • «Вид услуги» - вид услуги, по которому произведен платеж (определяется по коду и наименования элемента справочника «Виды услуг»);
    • «Номенклатура услуги» - номенклатура услуги, по которой произведен платеж (определяется по коду и наименованию элемента справочника «Номенклатура»);
    • «Лицевой счет определяется по» - выбирается значение поля, по которому определяется лицевой счет в базе;
    • «Услуга определяется по» - выбирается значение поля, по которому определяется услуга в базе;
    • «Контроль дублирования оплат» - если данный флажок установлен, перед загрузкой каждого платежа проверяется наличие в информационной базе платежа, зарегистрированного этой же датой по этому же лицевому счету, на эту же сумму. В случае, если такой платеж обнаружен, загрузка текущего платежа не производится.
    • «Штрихкод»: значение штрихкода в соответствии с настройками, указанными в «Учетной политике (ЖКХ)».

    Вид операции«Загрузка из систем сбора показаний приборов учета»

    Список параметров для вида операции «Загрузка из систем сбора показаний приборов учета»:

    • «Лицевой счет» - лицевой счет (определяется по коду, наименованию либо идентификатору элемента справочника «Лицевые счета» в зависимости от выбранного значения поля «Лицевой счет определяется по»);
    • «Прибор учета» - счетчик, который может определяться в зависимости от значения настройки «Способ определения прибора учета»:

    1) «по значению поля реестра» - при выборе данной настройки прибор учета может определяться по коду, наименованию, идентификатору либо заводскому номеру элемента справочника «Счетчики» в зависимости от выбранного значения поля «Счетчик определяется по»;

    2) «по услуге» - при выборе данной настройки прибор учета может определяться в зависимости от значения настройки «Способ определения услуги»:

      • «по значению поля реестра» - при выборе данного варианта настройки определяется назначенная на текущий лицевой счет услуга по наименованию или по коду элемента справочника «Услуги».
      • «по ссылке на услугу» - при выборе данного варианта настройки прибор учета определяется поуслуге, указанной в поле «Ссылка на услугу показания».

    Если по текущей услуге на один лицевой счет установлено несколько приборов учета, поиск нужного прибора среди всех установленных будет осуществляться по его порядковому номеру, который небоходимо указать в поле «Порядковый номер ПУ». Порядок определяется сортировкой приборов учета по наименованию от А до Я.

    • «Дата» - дата показаний счетчика, которая определяется по следующим настройкам:

    1) «Символ-разделитель даты» - выбирается разделитель частей даты. Возможен выбор следующих значений: «. (точка)», «, (запятая)», «; (точка с запятой)», «- (тире)», «/ (слеш)», «Без разделителя». Например, в дате реестра «03.02.2015» в качестве разделителя используется символ «. (точка)».

    2) «Порядок расположения параметров даты» - настройка представляет собой таблицу со строками, соответствующие частям даты: «День», «Месяц», «Год». При помощи кнопок «Переместить вверх» и «Переместить винз» можно изменить порядок состава даты показания в реестре. Например: для даты показания реестра «03.20.2015» соответствует следующий порядок в таблице: «Месяц», «День», «Год».

    • «Показание дневное» - показание счетчика дневное;
    • «Показание ночное» - показание счетчика ночное;
    • «Показание пиковое» - показание счетчика пиковое.

    Для параметров «Показание дневное», «Показание ночное», «Показание пиковое» имеется настройка «Символ-разделитель робной части», позволяющая определить количество знаков дробной части после разделителя. Доступны следующие значения для выбора: «, (запятая)», «. (точка)», «- (тире)», «Без разделителя». Настройки каждого параметра прибора учета и свойств параметров можно настраивать индивидуально.

    Для вида операции «Загрузка из систем сбора показаний приборов учета» имеется возможность настраивать загрузку сведений о показаниях приборов учета из файлов, в которых в каждой строке/теге указаны сведения по нескольким приборам учета. Для этого на форме элемента справочника расположены следующие элементы:

    • «Количество счетчиков в одной строке/тэге» - поле, в котором указывается значение, соответствующее количествам приборов учета в одной строке/теге файла загрузки. Кнопками регулирования поля добавляется (при нажатии на кнопку регулирования «Вверх») или удаляется последний в списке (при нажатии на кнопку регулирования «Вниз») параметр, соответствующий прибору учета. В списке параметров должно быть не менее одного прибора учета.
    • «Добавить прибор учета» - кнопка, по нажатию которой в список добавляется параметры, соответствующие прибору учета. При этом настройки параметров копируются от последнего в списке параметра прибора учета.
    • «Удалить прибор учета» - кнопка, по нажатию которой в списке удаляется текущий параметр, соответствующий прибору учета.

    Вид операции«Универсальная выгрузка данных»

    На форме элемента справочника расположены следующие настройки:

    • «Заголовок в файле выгрузки» - настройка определяет, будет ли выведен текст заголовка в поле "Текст заголовка" в начало файла выгрузки. Настройка доступна для Вида загрузки/выгрузки "Универсальная выгрузка данных", для формата файла "TXT". Доступны следующие значения: "Выводить", "Не выводить". При этом:
      • Если выбрано "Выводить", то в файле выводится текст заголовка с итоговыми показателями.
      • Если выбрано "Не выводить", то в файле не выводится текст заголовка.
    • «Текст заголовка» - поле для ввода текста, который выводится в начале файла выгрузки. Поле доступно для Вида загрузки/выгрузки "Универсальная выгрузка данных", для формата файла "TXT", при установленной настройке "Заголовок в файле выгрузки" в значение "Выводить". Имеется возможность в данном поле вставлять параметры, которые будут заменены на соответствующие значения при выгрузке данных в файл. Для вставки параметров в текст данного поля из списка либо нажмите правую кнопку мыши и выберите команду "Выбрать параметр заголовка", либо введите вручную. Доступно использование в тексте следующих параметров:
      • [КоличествоЗаписей] - параметр будет заменен на количество записей с данными по лицевым счетам.
      • [ОбщаяСуммаДолга] - параметр будет заменен на общую сумму долга по лицевым счетам.
      • [ОбщаяСуммаНачислений] - параметр будет заменен на общую сумму начислений по лицевым счетам.
      • [ПериодВыгрузки] - параметр будет заменен на период выгрузки в формате "01.01.2017".

    Список параметров для вида операции «Универсальная выгрузка данных»:

    • «Лицевой счет» - лицевой счет (определяется по наименованию либо идентификатору элемента справочника «Лицевые счета» в зависимости от выбранного значения поля «Поле идентификации л/с»);
    • «ФИО ответственного» - ответственный собственник наниматель лицевого счета;
    • «Фамилия жильца / Наименование юр. лица» - выгружается по следующим правилам:
      1. Если в качестве ответственного собственника выступает жилец из справочника "Жильцы", тогда в качестве фамилии выгружается значение поля "Фамилия" регистра сведений "Фамилия, имя, отчество физического лица".
      2. Если в качестве ответственного собственника выступает контрагент, тогда в текущей колонке выгружается значение поля справочника "Контрагенты" по правилам:
      - Если вид контрагента "Физическое лицо", то выгружается фамилия контрагента.
      - Для остальных видов контрагентов выгружается наименование контрагента.
    • «Имя» - выгружается по следующим правилам:
      1. Если в качестве ответственного собственника выступает жилец из справочника "Жильцы", тогда в качестве имени выгружается значение поля "Имя" регистра сведений "Фамилия, имя, отчество физического лица".
      2. Если в качестве ответственного собственника выступает контрагент, тогда в текущей колонке выгружается значение поля справочника "Контрагенты" по правилам:
      - Если вид контрагента "Физическое лицо", то выгружается имя контрагента.
      - Для остальных видов контрагентов поле "Имя" не заполняется.
    • «Отчество» - выгружается по следующим правилам:
      1. Если в качестве ответственного собственника выступает жилец из справочника "Жильцы", тогда в качестве отчества выгружаем значение поля "Отчество" регистра сведений "Фамилия, имя, отчество физического лица".
      2. Если в качестве ответственного собственника выступает контрагент, тогда в текущей колонке выгружается значение поля справочника "Контрагенты" по правилам:
      - Если вид контрагента "Физическое лицо", то выгружается отчество контрагента.
      - Для остальных видов контрагентов поле "Отчество" не заполняется.
    • «Адрес» - адрес лицевого счета (определяется согласно настройкам Учетной политики (ЖКХ));
    • «Сумма долга» - сумма задолженности на период, указанный в настройке «Период долга»;
    • «Площадь» - площадь помещения с учетом или без учета доли лицевого счета.
    • «Номер телефона» - номер телефона лицевого счета. Если номер телефона лицевого счета не указан, то выгружается номер телефона ответственного собственника, нанимателя данного лицевого счета.
    • «Сумма начислений» - сумма начислений по услугам в отчетном периоде.
    • «Период оплаты» - период выгрузки.
    • «Дата операции» - текущая системная дата - дата выполнения выгрузки данных - в формате "ДД.ММ.ГГГГ" (например, 26.09.2017).
    • «Расчетный счет организации» - основной расчетный счет организации.
    • «Пустое поле» - пустое значение.
    • «Счетчики и услуги» - блок сведений по счетчикам, либо по услугам соглано значению настройки "Выгружать сведения по", а именно:
      - Для значения "Счетчикам" выгружаются сведения по приборам учета согласно составу параметров блока сведений. Если в блоке сведений имеется параметр "Показание", тогда выгружаются последние переданные в месяце выгрузки показания счетчиков, установленных на лицевые счета или на соответствующие им помещения. Если счетчик многотарифный, то каждый блок сведений выводится в файле выгрузки в соответствии с тарифностью счетчика. При этом к значению параметра "Наименование" через пробел добавляется текст в соответствии виду тарифа показания: "ДЕНЬ" - для дневного показания, "НОЧЬ" - для ночного показания, "ПИК" - для пикового показания.
      - Для значения "Услугам" выгружаются сведения по услугам согласно составу параметров блока сведений. Если в блоке сведений имеется параметр "Сумма долга", тогда выгружается конечное сальдо по услугам в периоде выгрузки данных.
    • «ОКТМО» - Классификатор территорий из раздела "Налоговая инспекция" справочника «Организации»
    • «КБК» - Код бюджетной классификации. Определяется виду налога из справочника «Налоги и взносы»

    Символ-разделитель после каждого значения параметра выгрузки в файл формата "TXT" указывается в колонке "Разделитель после значения" таблицы с параметрами. Если данный символ совпадает с разделителем колонок на закладке "Настройки", тогда в колонке выводится значение "По умолчанию".

    Можно выгружать параметры счетчика ("Заводской номер", "Идентификатор", "Код", "Наименование", "Номер счетчика в ГИС ЖКХ") в отдельных колонках, либо через символ-разделитель в одной строке сведений по лицевым счетам. При этом параметр "Счетчик и показание" переименован в "Счетчики и услуги", и для данного параметра изменён состав настроек. Для выгрузки сведений о показаниях приборов учета необходимо значение настройки "Выгружать сведения по" изменить на значение "Счетчикам".

    Можно выгружать сведения по услугам ("Код услуги", "Наименование услуги", "Сумма долга") в раздельных колонках, либо через символ-разделитель. Для выгрузки сведений по услугам необходимо добавить параметр "Счетчики и услуги" и значение настройки "Выгружать сведения по" изменить на значение "Счетчикам".
