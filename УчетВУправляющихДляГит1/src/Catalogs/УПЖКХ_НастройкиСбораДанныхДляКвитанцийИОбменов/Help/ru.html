Справочник «Настройки сбора данных для квитанций и обменов»

Данный справочник предназначен для указания настроек формирования отчета "Платежный документ" и определения состава данных, выгружаемых на сайт ЖКХ в рамках варианта обмена "Выгрузка данных по ЛС".

На форме справочника располагаются следующие элементы:

  • «Наименование» — наименование настройки;
  • «Исполнитель» – указывается организация-исполнитель услуг;
  • Вкладка «Основные настройки» – хранит общие настройки вывода показателей в платежных документах и включает в себя:
    • Группа «Общие показатели и объекты»:
      • Флаг «Использовать ШК» – устанавливает факт необходимости формирования штрихкода;
      • Список выбора «Услуги исполнителя» – данная настройка состоит из следующих пунктов:
        1. «Выводить список услуг» – в качестве услуг организации-получателя платежа будут использоваться все услуги, для которых не созданы документы «Управления договорами с поставщиками»;
        2. «Выводить произвольную строку» – в качестве услуг организации-получателя платежа будет использоваться информация, вписанная пользователем.
      • Список выбора «Комиссия банка» – предназначен для выбора варианта учета комиссии за услуги банка и состоит из следующих пунктов:
        1. «Не учитывать» - комиссия банка не учитывается;
        2. «Сверх суммы к оплате» - комиссия рассчитывается по следующей формуле:
          [Комиссия] = [Итого к оплате] * [Процент комиссии] / 100
        3. «В сумме к оплате» - комиссия рассчитывается по следующей формуле:
          [Комиссия] = [Итого к оплате] * ([Процент комиссии] / 100 - [Процент комиссии])
      • Поле ввода «Процентная ставка» – размер процентной ставки за пользование услугами банка;
      • Флаг «Вместо общей площади использовать расчетную площадь» – устанавливает факт использования расчетных площадей помещения и здания вместо общих площадей;
      • Поле ввода «Расчетная площадь здания» – тип площади здания, размер которого необходимо использовать вместо размера общей площади;
      • Поле ввода «Расчетная площадь помещения» – вид площади помещения, размер которого необходимо использовать вместо размера общей площади;
      • Флаг «Учитывать начальное сальдо и оплаты» – устанавливает факт необходимости учета информации о начальном сальдо и об оплатах;
      • Список выбора «В сумму к оплате» – предназначен для выбора варианта включения начального сальдо и оплат в итоговую сумму к оплате. Состоит из следующих пунктов:
        1. «Не включать начальное сальдо и оплаты» – начальное сальдо и оплаты будут учтены информативно, без включения в итоговую сумму к оплате;
        2. «Включать задолженность и оплаты» – в итоговой сумме к оплате будут учтены только задолженность (положительное сальдо) и оплаты;
        3. «Включать аванс и оплаты» – в итоговой сумме к оплате будут учтены только аванс (отрицательное сальдо) и оплаты;
        4. «Включать задолженность, аванс и оплаты» – в итоговой сумме к оплате будут учтены начальное сальдо (и задолженность и аванс) и оплаты.
      • Список выбора «Учитывать все оплаты, поступившие до» – предназначен для выбора варианта учета поступивших оплат и состоит из следующих пунктов:
        1. «Конца отчетного месяца» – будут учтены все оплаты, поступившие до конца отчетного месяца;
        2. «Текущего момента» – будут учтены все оплаты, поступившие до даты формирования платежного документа.
      • Флаг «Учитывать объемы расхождения» – устанавливает факт необходимости сбора информации об объемах расхождений ОДН: учетный объем, объем расхождения.
      • Флаг «Выводить отрицательную сумму к оплате» - признак вывода отрицательного сальдо;
    • Группа «Показатели таблицы начисления»:
      • Флаг «Заполнять колонку инд. потребления для всех услуг» – устанавливает факт заполнения колонки с индивидуальным потреблением в таблице начисления для всех услуг;
  • Вкладка «Выбор услуг» – отвечает за список услуг, сведения по которым будут использованы при сборе и компоновке данных для обменов, и на который формируется платежный документ, и включает в себя:
    • Список выбора «Учитывать услуги» – предназначен для выбора одного из следующих трех пунктов:
      • «Все услуги» – сбор данных будет осуществляться по всем услугам, по которым имели место начисления в отчетном месяце;
      • «Выбранные услуги» – сбор данных будет осуществляться по услугам, выбранным пользователем;
      • «Все услуги, кроме выбранных» – сбор данных будет осуществляться по всем услугам, кроме выбранных пользователем.
    • Табличную часть, предназначенную для выбора услуг.
  • Вкладка «Порядок услуг» – отвечает за сортировку собранных данных, предназначенных для обменов и вывода сведений по услугам в квитанции, и включает в себя табличную часть, предназначенную для выбора видов услуг;
  • Вкладка «Разделение услуг» – отвечает за разделение услуг на три типа: «Коммунальные», «Содержание и ремонт жилья (Жилищные)», «Прочие», - и включает в себя:
    • Флаг «Разделять услуги по типам» – устанавливает факт вывода услуг с разделением по типам;
    • Табличную часть, предназначенную для установления соответствия между видами услуг и типами услуг.
      Примечание: для платежного документа «Плат. документ (с полями для отметки банка №2)» данная настройка не используется. 
  • Вкладка «Услуги без ОДН» – отвечает за выбор услуг, для которых необходимо отражать начисления за ОДН в составе индивидуальных начислений, и включает в себя табличную часть, предназначенную для выбора услуг.
  • Вкладка «Кап. ремонт» – хранит настройки сбора данных по услугам капитального ремонта для обменов и вывода отдельной квитанции в платежных документах, и включает в себя:
    • Список выбора «Печатать» – предназначен для выбора варианта учета сведений по услугам кап.ремонта, состоящий из следующих пунктов:
      1. «основную квитанцию и квитанцию на кап. ремонт» - сбор данных по основным услугам и услугам кап.ремонта осуществляется раздельно. При печати квитанций будут формироваться 2 квитанции - основная и квитанция по услугам кап.ремонта;
      2. «только квитанцию на кап. ремонт» - осуществляется только сбор данных по услугам кап.ремонта. При печати квитанций будет формироваться только квитанция по услугам кап.ремонта.
      3. «только основную квитанцию» - осуществляется только сбор данных по основным услугам. При печати квитанций будет формироваться только квитанция по основным услугам.
      4. «единая квитанция на осн. услуги и кап. ремонт» - будет осуществляться сбор общих данных по основным услугам и услугам кап.ремонта. При печати квитанций будет формироваться единая квитанция по основным услугам и услугам кап.ремонта.
  • Вкладка «Пени» – хранит настройки сбора данных по пеням для обменов и вывода в платежных документах. Включает в себя:
    • Флаг «Учитывать пени» – устанавливает факт учета пеней в начислениях;
    • Список выбора «Выводить пени» – доступен при установленом флажке «Учитывать пени» и предназначен для выбора варианта учета пени при сборе данных, состоящий из следующих пунктов:
      1. «в квитанции с основными начислениями» - данные о пенях при сборе данных будут учтены в рамках основных начислений. При печати квитанций сведения о пенях будут выводиться в одной квитанции с начислениями;
      2. «в отдельной квитанции» - данные о пенях при сборе данных будут учтены отдельно от основных начислений. При печати квитанций сведения о пенях будут выводиться в отдельной квитанции.
    • ВАЖНО: при выгрузке платежных документов в ГИС ЖКХ значение, выбранное в списке выбора «Выводить пени», не учитывается. При этом сведения о начислениях по пеням выгружаются всегда, когда флаг «Учитывать пени» установлен.
  • Вкладка «Настройки вывода квитанций» – хранит общие настройки вывода платежных документов и включает в себя:
    • Группа «Двусторонняя печать»:
      • Флаг «Использовать двустороннюю печать» - признак двусторонней печати квитанций. 
        При установленном флаге формирование квитанций производится на двух страницах. При этом, если содержимое квитанции умещается на лицевой (нечетной) странице, то выводится компенсирующая (пустая) оборотная (четная) страница (см. Рис. 1). Иначе, если содержимое квитанции не умещается на лицевой (нечетной) странице, вывод продолжается на оборотной (четной) странице (см. Рис. 2а);
      • Флаг «Выводить отступ при двусторонней печати» - признак вывода отступа, равного области извещения, на оборотной (четной) странице при двусторонней печати (см. Рис. 2б). Данная настройка доступна при установленном флаге «Использовать двустороннюю печать».
        Примечание: для платежных документов: «Плат. документ (приказ №454 в соответ. с пост. №354)» и «Плат. документ (Пермский край)», - данная настройка не используется;

    • Группа «Настройки печати»:
      • Поле «Отбор квитанций» – настройка позволяет задать отбор квитанций для вывода на печать по следующим параметрам:
        • «Печатать все квитанции» – выводятся все квитанции;
        • «Не печатать «нулевые» квитанции» – не выводятся квитанции, в которых итоговая сумма к оплате и сумма начисления равны нулю;
        • «Печатать квитанции с итоговой суммой в интервале» – выводятся квитанции, в которых итоговая сумма к оплате находится в указанном интервале в полях:
          • «от» – указывается левая граница интервала отбора;
          • «до» – указывается правая граница интервала отбора.
      • Флаг «Всегда выводить дополнительные области» – устанавливает факт вывода шапок "пустых" таблиц (таблиц, в которых не имеется данных);
      • Список выбора «Вариант сортировки квитанции» – предназначен для выбора варианта сортировки квитанций при групповой печати и состоит из следующих пунктов:
        1. «Адрес регистрации» – квитанции будут отсортированы в порядке возрастания номеров квартир, указанных в адресе регистрации;
        2. «Адрес доставки» - квитанции будут отсортированы в порядке возрастания номеров квартир, указанных в адресе доставки;
        3. «Лицевой счет» – квитанции будут отсортированы в порядке возрастания номеров лицевых счетов.
      • Флаг «Печатать отрывной «корешок» – устанавливает факт вывода отрывной части квитанций с информацией о последних показаниях индивидуальных приборов учета.
        Примечание: для платежного документа «Плат. документ (пост. №1161/57 для МО)» и «Плат. документ (пост. №679/30 для МО)» печать отрывного корешка не используется.
      • При установленном флаге «Печатать отрывной «корешок» доступен список выбора варианта расположения отрывного корешка, который состоит из следующих пунктов:
        1. «вверху квитанции» – отрывной корешок будет выводиться в верхней части квитанции;
        2. «внизу квитанции» – отрывной корешок будет выводиться в нижней части квитанции.
        Примечание: при двусторонней печати выбор варианта расположения отрывного корешка не доступен, отрывной корешок в таком случае будет выводиться только в верхней части квитанции.
    • Группа «Плат. документ (Пермский край)»:
      • Флаг «Не выводить показания по ПУ» - признак отказа вывода блока с показаниями по ПУ для отчета "Плат. документ (Пермский край)".
        Примечание: данная настройка используется только для платежного документа «Плат. документ (Пермский край)».
  • Вкладка «Услуги без ОДН» – отвечает за выбор услуг, для которых необходимо отражать начисления за ОДН в составе индивидуальных начислений, и включает в себя табличную часть, предназначенную для выбора услуг.
  • Вкладка «Кап. ремонт» – хранит настройки сбора данных по услугам капитального ремонта для обменов и вывода отдельной квитанции в платежных документах, и включает в себя:
    • Список выбора «Формировать» – предназначен для выбора варианта учета сведений по услугам кап.ремонта, состоящий из следующих пунктов:
      1. «основную квитанцию и квитанцию на кап. ремонт» - сбор данных по основным услугам и услугам кап.ремонта осуществляется раздельно. При печати квитанций будут формироваться 2 квитанции - основная и квитанция по услугам кап.ремонта;
      2. «только квитанцию на кап. ремонт» - осуществляется только сбор данных по услугам кап.ремонта. При печати квитанций будет формироваться только квитанция по услугам кап.ремонта.
      3. «только основную квитанцию» - осуществляется только сбор данных по основным услугам. При печати квитанций будет формироваться только квитанция по основным услугам.
      4. «единая квитанция на осн. услуги и кап. ремонт» - будет осуществляться сбор общих данных по основным услугам и услугам кап.ремонта. При печати квитанций будет формироваться единая квитанция по основным услугам и услугам кап.ремонта.
  • Вкладка «Пени» – хранит настройки сбора данных по пеням для обменов и вывода в платежных документах. Включает в себя:
    • Флаг «Учитывать пени» – устанавливает факт учета пеней в начислениях;
    • Список выбора «Выводить пени» – доступен при установленном флажке «Учитывать пени» и предназначен для выбора варианта учета пени при сборе данных, состоящий из следующих пунктов:
      1. «в квитанции с основными начислениями» - данные о пенях при сборе данных будут учтены в рамках основных начислений. При печати квитанций сведения о пенях будут выводиться в одной квитанции с начислениями;
      2. «в отдельной квитанции» - данные о пенях при сборе данных будут учтены отдельно от основных начислений. При печати квитанций сведения о пенях будут выводиться в отдельной квитанции.
    • ВАЖНО: при выгрузке платежных документов в ГИС ЖКХ значение, выбранное в списке выбора «Выводить пени», не учитывается. При этом сведения о начислениях по пеням выгружаются всегда, когда флаг «Учитывать пени» установлен.
