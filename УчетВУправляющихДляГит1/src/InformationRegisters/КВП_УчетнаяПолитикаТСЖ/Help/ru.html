Регистр сведений «Учетная политика (ЖКХ)»

Данный регистр предназначен для хранения настроек учетной политики управляющих компаний ЖКХ, ТСЖ и ЖСК. Регистр имеет периодичность в пределах месяца. Каждая запись этого регистра характеризует учетную политику организации за определенный период.

При вводе новой записи об учетной политике необходимо выбрать организацию и дату начала отчетного периода, в котором будет применяться данная учетная политика.

Закладка «Общее»

На этой закладке указывается информация:

  • «Срок оплаты» – срок, по истечении которого возможно начисление пени.
  • «Учитывать периоды с разным количеством проживающих и зарегистрированных» – определяет, будут ли учитываться периоды с разным количеством проживающих и зарегистрированных при начислении услуг.
  • «Период приема показаний» – период каждого месяца, в течение которого в мобильном приложении «ЖКХ: Личный кабинет» и на Сайте ЖКХ ввод показаний счетчиков доступен для жильцов. При этом имеется возможность указания причин ограничения периода ввода показаний, которые будут переданы жильцам.

 

Закладка «Паспортный стол»

На этой закладке имеется возможность задать следующие параметры:

  • «Ответственное лицо» - указывается ответственное лицо паспортного стола, ФИО которого будет автоматически подставляться в формы паспортного стола:
    • Ответственное лицо организации - выбирается должность ответственного лица в организации. По указанной должности будет автоматически определяться ответственное лицо по данным, указанным в регистре сведений «Ответственные лица организаций».
    • Сотрудник организации - выбирается работник из справочника «Сотрудники организаций».
    • Текущий пользователь - в качестве ответственного лица будет выбран текущий пользователь.
  • «Орган регистрационного учета» - указывается орган учета, наименование которого будет автоматически подставляться в формы паспортного стола, формируемые с помощью отчета «Формы регистрационного учета» на формах №1,3,6,8,14.

Закладка «Регистрация оплат»

На этой закладке указываются параметры распределения оплат по услугам:

  • «Распределять оплаты» - может принимать следующие значения:
    • «В первую очередь по начислениям за услуги, затем по долгам» - при указании суммы оплаты либо нажатии на кнопку «Распределить» в документе «Регистрация оплаты» сумма оплаты разносится по услугам, по которым было произведено начисление в прошлом месяце в размере этого начисления. Остальная сумма идет на погашение долгов.
    • «В первую очередь по долгам, затем по начислениям за услуги» - при указании суммы оплаты либо нажатии на кнопку «Распределить» в документе «Регистрация оплаты» сумма оплаты распределяется на погашение задолженности. Остальная сумма идет на погашение начислений, произведенных в прошлом месяце.
    • «Пропорционально по долгам и начислениям за услуги» - При указании суммы оплаты либо нажатии на кнопку «Распределить» в документе «Регистрация оплаты» сумма оплаты распределяется на погашение задолженности и начислений, произведенных в прошлом месяце, пропорционально суммам задолженности и начислений.
  • «Распределять аванс» - если для распределения нет основания, сумма распределяется на все назначенные для лицевого счета услуги следующими способами:
    • «Поровну по действующим услугам» - сумма будет поделена поровну между всеми действующими услугами лицевого счета;
    • «Пропорционально начислениям» - сумма будет поделена между действующими услугами пропорционально начислениям за указанное количество прошлых месяцев (начиная с месяца, предшествующего дате документа оплаты).
  • «Учитывать месяц начисления при считывании оплаты со штрихкода» - если установить данный флажок, то оплата, регистрируемая через штрихкод, в первую очередь попадет на месяц, указанный в штрихкоде. Если же оплата перекроет сумму начисления за данный месяц, то разница будет распределена на другие месяцы согласно соответствующим настройкам.
  • «Порядок услуг для распределения оплат» - в табличной части указываются виды услуг, по которым будет выполняться приоритетное погашение задолженности. Распределение оплаты на указанные виды услуг будет происходить в соответствии с заданным в табличной части порядком.

Закладка «Пени»

На этой закладке указываются настройки для расчета пени:

  • «Настройка расчета пени» - указывается настройка из справочника «Настройки расчета пени».
  • «Использовать список услуг-исключений» - если флажок установлен, то по указанным в регистре сведений «Услуги-исключения для начисления пени» услугам в разрезе текущей организации не будет производиться начисление пени. Изменить список услуг-исключений можно по нажатию кнопки «Указать список услуг-исключений».

Закладка «Льготы»

На этой закладке указываются настройки для расчета льгот:

  • «Задать настройки предоставления и расчета льгот» - ссылка на настройки предоставления и расчета льгот. При нажатии на ссылку открывается новое окно, в котором фиксируются настройки предоставления и расчета льгот в разрезе групп услуг. Это позволяет для разных услуг установить различные настройки расчета льгот.
  • «Не предоставлять льготу:»
    • на услуги, рассчитываемые по кол-ву проживающих, если льготник зарегистрирован, но не проживает, а льгота предоставляется зарегистрированным жильцам(*);
    • на услуги, рассчитываемые по кол-ву зарегистрированных, если льготник проживает, но не зарегистрирован, а льгота предоставляется проживающим жильцам(**);
    • на услуги, рассчитываемые по количеству проживающих / зарегистрированных, если льготник является собственником, но не проживает / не зарегистрирован, а льгота предоставляется собственникам(***).

    В ситуациях (*), (**) и (***) льгота рассчитывается только на услуги со следующими способами расчета:

    • по площади;
    • с ограничением по норме, в состав которых входят услуги, рассчитываемые по площади;
    • по показаниям счетчика (для начислений, произведенных только по показаниям приборов учета).

    Примечание

    Вариант предоставления льгот жильцам (зарегистрированным, проживающим или собственникам) устанавливается в «Настройках предоставления и расчета льгот».

  • «Не учитывать льготы во взаиморасчетах» - если данный флажок установлен, то при расчете льгот суммы начислений на лицевые счета не уменьшаются (льгота рассчитывается справочно).
  • «При расчете льгот использовать льготные тарифы» - если флажок установлен, то расчет льгот для указанных услуг производится по льготным тарифам. Настройка работает в совокупности со следующей настройкой:
    • «Задать список услуг для расчета льгот по льготному тарифу» - ссылка на настройку услуг, по которым необходимо рассчитывать льготы по тарифам, отличным от тарифов начисления. Ссылка доступна при установленном флажке «При расчете льгот использовать льготные тарифы». Размеры льготных тарифов устанавливаются в документе «Установка тарифов».

Закладка «Перерасчет»

На этой закладке указываются параметры перерасчета начислений по услугам:

  • «Список услуг, не подлежащих перерасчету» - указывается список услуг, начисления которых не подлежат перерасчету. По указанным в списке услугам не будет производиться автоматический перерасчет в документе «Корректировка начислений».

Закладка «Закрытие периода»

На этой закладке указываются параметры закрытия периода по начислениям ЖКХ. При распределении авансов, сначала аванс по услуге распределяется по долгам этой же услуги по всем месяцам задолженности. Нераспределенные остатки распределяются в зависимости от следующих настроек:

  • «Разрешить перенос авансов с одной услуги на долги по другим услугам» - если флажок установлен, то при закрытии периода аванс по одной услуге будет распределяться по долгам других услуг. В противном случае аванс по одной услуге будет всегда распределяться только на долги этой же услуги.
  • «Выполнять перенос в рамках одного месяца начисления» (данный флажок доступен, если установлен флажок «Разрешить перенос авансов с одной услуги на долги по другим услугам») - если флажок установлен, то аванс по услуге будет распределяться по долгам других услуг только в пределах одного месяца начисления. В противном случае распределение будет производиться по всем месяцам задолженности по методу ФИФО.
  • «Разрешить перенос аванса по начислениям на погашение задолженности по пени» - если флажок установлен, то аванс по начислениям услуг будет распределяться по задолженности начислений и пени согласно варианту оплаты пени (вариант оплаты пени, указанный на закладке «Пени»). В противном случае аванс по начислениям на задолженность по пени распределяться не будет.

Закладка «ТСЖ»

На этой закладке указывается следующая информация:

  • «Председатель ТСЖ» – из справочника «Физические лица» выбирается председатель ТСЖ, ФИО которого будет выводиться в печатной форме документа «Смета ТСЖ».
  • «Источник финансирования (взносы)» - из справочника «Назначения целевых средств» выбирается источник финансирования, соответствующий членским взносам.
  • «Учитывать вступительные взносы ТСЖ» - если данный флажок будет установлен, то при формировании документа «Вступление в ТСЖ» можно задать размер вступительного взноса для каждого члена ТСЖ и автоматически сформировать документ «Приходный кассовый ордер».
  • «Суммы доходов определять автоматически» - если флажок установлен, то при автоматическом формировании документа «Приходный кассовый ордер» из документа «Вступление в ТСЖ» суммы доходов определяются автоматически. Если флажок не установлен, то можно указать, необходимо ли обнулять принимаемые доходы (флажок "Обнулять принимаемые доходы").

Закладка «УСН»

Данная закладка видна только для организаций с упрощенной системой налогообложения. На этой закладке указываются следующие настройки:

  • «Формировать счета-фактуры» – если данный флажок установлен, то при проведении документа «Отражение начислений в регламентированном учете» счета-фактуры формируются автоматически.
  • «Автоматически отражать оплату начислений в КУДиР» - если флажок установлен, то при автоматическом формировании документов «Поступление наличных», «Выдача наличных», «Поступление на расчетный счет» и «Списание с расчетного счета» (далее - Типовые документы оплаты) из документа «Регистрация оплаты» суммы доходов определяются автоматически. Если флажок не установлен, то можно указать какие из доходов (Всего и/или Принимаемые) будут обнуляться.

    Имеется возможность настройки отражения доходов в разрезе услуг. Для этого необходимо установить переключатель «Отражать оплату начислений в КУДиР» в значение «В разрезе услуг». Если установлен данный вариант отражения, то при проведении Типовых документов оплаты, указанных в качестве документов оплаты в табличной части документа «Регистрация оплаты», автоматически будут формироваться записи в книгу учета доходов и расходов только по тем услугам, которые заданы в настройке оплаты в КУДиР.

    При отражении оплат настройки учитываются по следующим приоритетам:

    1. Настройки, указанные для конкретной услуги;
    2. Настройки, указанные для группы услуг;
    3. Настройки, у которых поле «Услуга» не заполнено.

При выборе сводного отражения оплаты начислений в КУДиР или отражения оплаты начислений в КУДиР в разрезе услуг возможно дополнительно установить флажок «Доходы КУДиР от начисления пени всегда считать автоматически». При установке данного флажка доходы КУДиР от начисления пени считаются автоматически, даже если согласно сводным настройкам или индивидуальным настройкам отражения услуг доходы обнуляются. Если флажок не установлен, то отражение пени в КУДиР производится по сводным настройкам или индивидуальным настройкам соответствующих услуг.

Закладка «Регламентированный учет»

Для отражения начислений услуг в бухгалтерском и налоговом учете с помощью документа «Отражение начислений в регламентированном учете» нужно установить настройку «Отражать начисления в регламентированном учете». После этого на закладке станут доступны настройки отражения:

Примечание

При отражении начислений услуг в бухгалтерском и налоговом учете формируется проводка Дт СчетВзаиморасчетов Кт СчетДоходов.

  • «Отражение доходов от реализации услуг производить на»:
    • Счете учета доходов от реализации номенклатуры услуги - в кредит проводки, формируемой при отражении начислений, будет подставляться счет учета доходов от реализации из регистра сведений «Счета учета номенклатуры».
    • Счете учета номенклатуры услуги - в кредит проводки, формируемой при отражении начислений, будет подставляться счет учета номенклатуры из регистра сведений «Счета учета номенклатуры».
  • «Отражение расчетов с контрагентами за услуги капитального ремонта производить на»:
    • Счетах взаиморасчетов с контрагентами - в дебет проводки, формируемой при отражении начислений по услугам капитального ремонта, будет подставляться счет расчетов из регистра сведений «Счета расчетов с контрагентами».  
    • Отдельных счетах - в дебет проводки, формируемой при отражении начислений по услугам капитального ремонта, будет подставляться указанный счет. Данный счет будет также использоваться в качестве счета учета авансов по услугам капитального ремонта.
  • «Реализацию услуг при агентской схеме производить от имени»:
    • Принципала - при отражении в регламентированном учете счета-фактуры покупателям выставляться не будут.
    • Агента - при отражении в регламентированном учете будут выставляться счета-фактуры, при этом счета-фактуры не будут регистрироваться в книге продаж.
  • «Разрешить формирование документов регламентированного учета с отрицательными суммами» - при установленной настройке отрицательные суммы будут отражаться в регламентированном учете с помощью документа "Реализация товаров и услуг", независимо от выбранного типа документа для отражения.
  • «Способ отражения льгот»:
    • «Отдельно от начислений» - при сводном отражении (упрощенном) суммы начислений и суммы льгот будут отражаться в регламентированном учете отдельными проводками. При отражении документами регламентированного учета для льгот будут формироваться документы "Корректировка долга", а для начислений - "Акт об оказании производственных услуг" или "Реализация товаров и услуг".
    • «Уменьшением начислений» - сумма начислений будет уменьшаться на сумму льгот. При сводном отражении (упрощенном) будет формироваться одна проводка. При отражении документами регламентированного учета будут формироваться документы "Акт об оказании производственных услуг" или "Реализация товаров и услуг".
  • «Отражать пени отдельно от начислений» - при установленной настройке суммы пени будут отражаться в регламентированном учете отдельно от начислений услуг. При упрощенном отражении суммы начислений и пени будут отражаться в регламентированном учете отдельными проводками. При отражении документами регламентированного учета для пеней будут формироваться отдельные документы. Нажав на гиперссылку «Счета учета расчетов по пеням», можно указать счета учета, которые будут использоваться при отражении пеней.
  • «Отражать пени за капитальный ремонт отдельно от начислений» - при установленной настройке суммы пени за услуги капитального ремонта будут отражаться в регламентированном учете отдельно от начислений за услуги капитального ремонта. При упрощенном отражении суммы начислений и пени за услуги капитального ремонта будут отражаться в регламентированном учете отдельными проводками. При отражении документами регламентированного учета для пеней будут формироваться отдельные документы. Счета учета можно установить, нажав на гиперссылку «Счета учета расчетов по пеням за капитальный ремонт».

Примечание

Настройки «Отражение расчетов с контрагентами за услуги капитального ремонта производить на» и «Отражать пени за капитальный ремонт отдельно от начислений» доступны только при ведении раздельного учета по услугам капитального ремонта.

При ведении раздельного учета услуг капитального ремонта настройка «Отражать пени отдельно от начислений» влияет только на пени по услугам кроме капитального ремонта. Если необходимо отражать на отдельных счетах пени по услугам капитального ремонта, то следует установить настройку «Отражать пени за капитальный ремонт отдельно от начислений».

Если после изменения настроек отражения пеней или льгот необходимо, чтобы данные настройки учитывались в уже сформированных документах "Отражение начислений в регл. учете", то сначала нужно перепровести документы начислений, отражение которых выполнялось. После этого следует перезаполнить документы "Отражение начислений в регл. учете" и выполнить отражение начислений.
